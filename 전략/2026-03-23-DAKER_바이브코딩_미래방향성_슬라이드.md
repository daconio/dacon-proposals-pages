### 경진대회의 진화: 알고리즘에서 프로덕트로

#### AS-IS: 기존 데이콘 모델

- **데이터 경진대회**: CSV 데이터셋 → 모델 학습 → 리더보드 순위 경쟁
- **알고리즘 중심**: 정확도 0.001 차이로 승부. 실무 적용과 괴리.
- **개인 경쟁**: 혼자 노트북에서 작업. 팀워크/커뮤니케이션 검증 불가.

#### TO-BE: DAKER 바이브코딩 해커톤

- **프로덕트 빌딩 해커톤**: AI 도구로 **실제 작동하는 소프트웨어** 개발. 프롬프트 해커톤 지원.
- **바이브코딩 역량 중심**: AI를 활용한 **문제 해결 능력** 평가. 프레젠테이션 + 쇼케이스 기반.
- **팀 빌딩 + AI 매칭**: AI 기반 팀 구성, 그룹 관리, **실전 협업 역량** 검증.

### 고객의 솔직한 목소리

기존 데이콘 공모전 결과물에 대한 주최측(기업) 피드백 — 데이터 유니버스 사례

#### Pain Points — 기업이 느끼는 괴리

- ✗ **"결과물이 너무 이상적"**: 학술 논문 수준의 모델은 있지만 **실제 서비스에 바로 적용할 수 없음**. 데이터 전처리부터 배포까지의 간극이 큼.
- ✗ **"현실 데이터와 괴리"**: 깨끗한 CSV 기반 경진대회 vs **노이즈 가득한 현실 데이터**. 기업 현장의 데이터 품질, 보안, 인프라 제약을 경험하지 못한 결과물.
- ✗ **"운영/유지보수 관점 부재"**: 정확도 0.001 최적화에 집중하지만, **비용, 속도, 모니터링, 버전 관리** 등 프로덕션 관점은 전무.

#### Solution — DAKER가 해결하는 방법

- ✓ **프로덕트 빌딩 해커톤**: 모델이 아닌 **실제 작동하는 소프트웨어**를 만드는 대회. 바이브코딩으로 프로토타입부터 배포까지 경험.
- ✓ **기업 실제 과제 연동**: FDE 프로젝트 마켓을 통해 **기업의 실제 문제**를 해커톤 과제로 연결. 현실 데이터, 현실 제약 조건 하에서 문제 해결.
- ✓ **프레젠테이션 + 쇼케이스 평가**: 코드 정확도가 아닌 **문제 정의, 솔루션 실현 가능성, 비즈니스 임팩트**를 종합 평가. 기업이 바로 채택할 수 있는 결과물.

> "경진대회 수상작은 인상적이지만, 우리 현장에 적용하려면 처음부터 다시 개발해야 합니다. **이상과 현실의 괴리**를 좁힐 수 있는 방식이 필요합니다."
> — 데이터 유니버스 공모전 주최 기업 피드백
