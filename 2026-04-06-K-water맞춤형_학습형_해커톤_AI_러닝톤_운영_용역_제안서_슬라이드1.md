## Ⅰ. 과업의 이해

### Ⅰ-1. 과업목표 및 추진 목적

**비전공자도 완주할 수 있는 학습형 해커톤(러닝톤) 설계·운영**

- 🎯 **과업목표(한 줄 정의)**: "한국수자원공사 임직원이 **생성형 AI로 실제 업무를 바꿔보는 경험**을, **비전공자도 완주할 수 있는 학습형 해커톤**으로 설계·운영한다."

**3층 목표**

| 구분 | 내용 |
|---|---|
| 본 과업 | **학습(Learn) + 해커톤(Hackathon) = 러닝톤** 운영 용역 |
| 1차 목표 | 한국수자원공사 맞춤형 생성형 AI 교육 연계 · 실전 역량 내재화 |
| 2차 목표 | 약 30팀 90명 규모 아이디어 발굴·프로토타입 완성 |
| 3차 목표 | 우수 산출물의 업무 적용 파일럿 및 전사 확산 기반 |

**🧭 추진 목적**

- **AI First 전환의 현장 체화** — 전 임직원의 실무로 안착
- **업무 밀착형 역량 강화** — 바이브코딩·문서 자동화·RAG
- **비전공자 포용** — 코드 없이 시작 2-Track 커리큘럼
- **사내 과제 발굴·연계** — 현업 페인포인트 → 대회 과제
- **지속가능한 학습 문화** — 지식 자산 축적

### Ⅰ-2-가. 러닝톤 운영

**대회 기획·플랫폼 제작부터 모집·예선·하이브리드 평가·본선·사후관리까지 전 과정 턴키 수행**

**🏁 대회 전 주기 6단계**

| 단계 | 활동 | 산출물 |
|---|---|---|
| 1. 종합 기획 | 컨셉·규모·일정·과제 풀 설계 | 운영 기획서 |
| 2. 공고·신청 | 사내 공고·랜딩·팀 빌딩 | 참가 신청 DB |
| 3. 온라인 예선 | 아이디어 제출·상호평가·심사 | 예선 리더보드 |
| 4. 본선 진출팀 | 상위 30팀 선발·과제 확정 | 본선 명단 |
| 5. 본선 운영 | 현장 진행·발표·시상 | 본선 결과물 |
| 6. 사후 관리 | 결과 보고·데이터 이관 | 최종보고서 |

**🛠 통합 운영 플랫폼 (웹·설치 불필요)**
- 참가·팀 관리 — 셀프 가입·팀 빌딩·병합
- 과제 제출·버전 관리 — 드래그&드롭·버전 이력
- 심사 기준 설정·점수 입력·자동 집계
- 실시간 리더보드 — ELO + 심사 + 직원 투표 통합
- 공지·자료·알림 · 일정 캘린더 · Q&A 게시판

**🎯 운영 KPI**: 참가 신청 전환율 ≥ 25% · 본선 진출팀 30팀 · 완주율 ≥ 85%

### Ⅰ-2-나. 온라인 멘토링

**전문가와 AI의 하이브리드형 멘토링으로 러닝톤을 끝까지 지원합니다**

- 📅 **정기 세션**: 팀당 **3회** (화상·플랫폼 통합)
- ⚡ **수시 세션**: AI 멘토 **24/7** (야간·주말 공백 제거)
- 🏁 **현장 세션**: 본선 **1박 2일** 사람 멘토 상주
- 🎯 **4단계 코칭 역할**: 정의 → 설계 → 구현 → 고도화

**👥 멘토단 구성**

| 항목 | 내용 |
|---|---|
| 멘토 자격 | 생성형 AI 프로젝트 · 컨설팅·교육 · 업무 개선 중 **2개 이상 충족** |
| 분야 구성 | LLM · 바이브코딩 · 데이터 · 자동화 · UX **5축** |
| 평균 경력 | 8년 이상 |
| 배치 기준 | **1인당 최대 3팀** (과업지시서 준수) |

**📚 한국수자원공사 맞춤 4-모듈 커리큘럼**

| 모듈 | 주제 | 분량 |
|---|---|---|
| M1 | 생성형 AI 입문 | 60분 |
| M2 | 바이브코딩 (기업용 AI 서비스) | 120분 |
| M3 | 업무 자동화 (RAG·요약) | 90분 |
| M4 | 아이디어톤 가이드 | 30분 |

진도·체류시간·퀴즈 자동 집계 · 북마크·이어보기 · **수료증 자동 발급**

**🎯 멘토링 KPI**: 만족도 4.3/5.0+ · 수료율 90%+ · AI 역량 향상 +30%p

### Ⅰ-2-다. 러닝톤 현장 운영

**본선 1박 2일 현장을 설치부터 정시 시상까지 무사고로 완성**

**🏗 1박 2일 본선 운영 (7월 초)**

| 구분 | 주요 활동 |
|---|---|
| 1일차 오전 | 개회식 · 오리엔테이션 · 과제 브리핑 |
| 1일차 오후 | 집중 구현 · 중간 멘토링 (사람 + AI) |
| 1일차 야간 | 야간 작업(희망팀) · AI 멘토 24/7 |
| 2일차 오전 | 최종 구현 · 제출 마감 · 심사위원 및 참가자 상호투표 |
| 2일차 오후 | 결선 발표 평가 · 시상식 |

**🎪 현장 구성**
- 🎤 무대 (발주기관 제공 + 부족분 추가)
- 💻 개발 공간 (팀별 부스 30개 · 전기·통신)
- ☕ 편의시설 (휴게·다과 · 발주처 협의)
- 🛡 안전 (방재 책임자 · AED · 의료 · 단체보험)
- 🍱 위생(하계) — HACCP 식자재 · 2시간 소진 원칙

**🚨 돌발 대응**: 5분 내 현장 조치 · PM·발주처 에스컬레이션 · 발주기관 추천 인력 우선 고려

**🎯 현장 운영 KPI**: 만족도 4.5/5.0+ · 과제 제출율 ≥ 95% · 시상식 정시 진행

### Ⅰ-2-라. 성과 관리

**러닝톤 성과를 정량지표와 정성지표로 이중 측정**

**📊 정량지표**
- 리커트척도 만족도 — 5점 평균 **4.5 이상**
- 참여자 수 — 본선 30팀 90명 · 신청 전환율 25%+
- 과제 제출율 — 본선 제출 95% 이상
- 완주율·수료율 — 완주 85%+ · 수료 90%+
- 프로토타입 건수 — 본선 산출물 30건 이상

**💬 정성지표**
- 멘토링 만족도 조사 — 팀별 개방형 + 리커트 병행
- 교육효과 조사 — 사전·사후 역량 자가진단 비교
- 추천·재참여 의향 — 차기 기수 참여·동료 추천 의사
- 업무 적용 인터뷰 — 본선 +4주 현업 적용성 추적
- 개방형 피드백 — 대회 개선점·차기 기수 제안

**📐 측정 체계 — 3시점 타임라인**

| 시점 | 방법 |
|---|---|
| 사전 (Baseline) | 역량 자기진단 설문 · 사내 인지도 |
| 진행 중 (월간) | 플랫폼 자동 집계 · 중간 설문 |
| 사후 (+2주) | 만족도·추천의향 설문 · 멘토 심층면담 |
| 사후 (+4주) | 업무 적용 인터뷰 · 케이스 스터디 |

**💡 측정 차별화 제안**
- Baseline 사전 설문으로 **변화량 측정**
- **3중 교차 검증** — 참가자 주관 + 심사자 객관 + 현업 적용성
- **+4주 업무 적용 추적** — 단발 이벤트 아닌 실제 도입 확인
- **이상 패턴 자동 필터** — 부정 응답·중복 IP·시간 편향 자동 제거
