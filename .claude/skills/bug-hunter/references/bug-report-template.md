# 버그 수정 완료 리포트 템플릿

아래 형식을 기반으로 작성한다. 실제 수정 내용과 검증 결과를 정확하게 반영해야 한다.

---

```markdown
# {주제} 버그 수정 완료 리포트

> **작성일**: YYYY-MM-DD
> **테스트 환경**: vitest + Playwright E2E
> **수정 브랜치**: {브랜치명}

---

## 1. 한눈에 보기 (Executive Summary)

### 수정 전 (Before)
- **사용자 경험**: [버그로 인한 사용자 체감 문제]
- **영향 범위**: [영향 받는 기능/사용자]
- **비즈니스 리스크**: [장애 수준]

### 수정 후 (After)
- **사용자 경험**: [수정 후 정상 동작 설명]
- **검증 결과**: [단위 테스트 N개 통과, E2E N개 시나리오 통과]
- **잔여 리스크**: [있으면 기술, 없으면 "없음"]

### 핵심 지표

| 지표 | 수정 전 | 수정 후 |
|------|--------|--------|
| 버그 수 | N건 | 0건 |
| 테스트 수 | N개 | N+M개 |
| E2E 검증 통과 | - | N/N 시나리오 |

## 2. 버그별 수정 상세

**근본 원인**: [한 문장]
**수정 내용**: [구체적 코드 변경]
```
