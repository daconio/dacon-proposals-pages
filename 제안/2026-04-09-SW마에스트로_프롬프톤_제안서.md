### S3. 현장 라이브 운영 — 미션 크리티컬 접근

> **단 한 치의 오차도 허용되지 않는 현장 라이브 해커톤**

**왜 미션 크리티컬인가**

현장 라이브 해커톤은 온라인 대회와 본질적으로 다릅니다. 참가자 150명이 The K Hotel 한 공간에서 **정해진 2시간** 동안 동시에 경쟁하며, 네트워크 지연이나 시스템 오류가 발생했을 때 **시간을 연장하거나 유연하게 대처하기 어려운 물리적 한계**가 있습니다.

| 구분 | 온라인 해커톤 | **현장 라이브 해커톤** |
|------|---------------|------------------------|
| **시간 유연성** | 장애 시 시간 연장 가능 | **연장 불가** — 참가자 전원 현장 대기 중 |
| **트래픽 패턴** | 분산된 동시 접속 | **초단위 동시 스파이크** (전원 동시 시작·제출) |
| **장애 복구** | 재시도 안내로 수습 가능 | **수백 명의 현장 불만** 즉시 발생 |
| **성공 기준** | 평균 가용성 99% | **무결성 100% — 모든 제출 0 누락** |

**6대 미션 크리티컬 요건**

| # | 요건 | 내용 |
|---|------|------|
| 1 | **동시 트래픽 내성** | 150명 동시 접속·동시 제출 시점의 초단위 트래픽 스파이크를 견디는 서버 인프라 |
| 2 | **실시간 리더보드 무오차** | 모든 제출이 누락·중복·순서 오류 없이 실시간 리더보드에 반영 |
| 3 | **LLM API 연동 안정성** | ChatGPT API 등 외부 LLM 연동의 rate limit, 타임아웃, 429 오류 사전 방어 |
| 4 | **현장 네트워크 이중화** | The K Hotel Wi-Fi + 백업 회선(4G/5G 라우터) 병행 운영으로 네트워크 단일 장애점 제거 |
| 5 | **제로 데이터 손실** | 제출 직후 자동 스냅샷, 로컬 버퍼 + 서버 이중 저장으로 네트워크 일시 단절 시에도 제출 보존 |
| 6 | **현장 즉시 대응 인력** | 운영 PM 1명 + 현장 엔지니어 2명 + 온라인 백업 엔지니어 1명 상시 모니터링 |

**4대 장애 대응 시나리오 — 사전 리허설 필수**

| 시나리오 | 예상 장애 | 대응 전략 |
|----------|-----------|----------|
| **S1. 서버 다운** | 플랫폼 응답 없음 | 핫스탠바이 서버 즉시 전환, 모든 제출 서버 간 실시간 복제 |
| **S2. LLM API 장애** | ChatGPT API 429/500 | Fallback API Key 풀 전환, 재시도 큐 자동 처리 |
| **S3. 네트워크 단절** | 현장 Wi-Fi 다운 | 백업 회선 자동 전환 + 참가자 로컬 저장 기능 발동 |
| **S4. 채점 오류** | 리더보드 점수 불일치 | 채점 엔진 이중화, 5분 간격 자동 정합성 검증, 불일치 시 즉시 알림 |

**사전 테스트 — 실 운영 환경 동일 부하 테스트 의무 실시**

| 회차 | 목적 | 확인 항목 | 필수 여부 |
|------|------|----------|----------|
| 1차 | 데이터 무결성 | 결측값, 포맷 오류, 가명화 누락 | **필수** |
| 2차 | 평가 산식 검증 | 점수 범위 정상화, 엣지케이스 처리 | **필수** |
| 3차 | 플랫폼 연동 | 제출→채점→리더보드 전체 흐름 | **필수** |
| 4차 | **부하 테스트 (150명 동시)** | 동시 접속·동시 제출 시 성능 및 안정성, LLM API 동시 호출 내성 | **필수** |
| 5차 | **최종 리허설 (The K Hotel 현장)** | 실제 행사장 네트워크 환경에서 전체 흐름 점검, 장애 시나리오 훈련 | **필수** |

### S4. 플랫폼 운영

| **갤러리 · 진행 현황 대시보드** | 개인/팀별 제출 현황과 점수를 실시간으로 공유. 전체 공개 현황과 운영 전용 현황 분리 운영. 순위 변동 알림 제공 |

### S5. DAKER 플랫폼 연동 페이지

> **본 프롬프톤은 DAKER 플랫폼의 해커톤·팀빌딩·그룹 결성 페이지와 유기적으로 연동됩니다.**
> 연수생은 단일 플랫폼 안에서 팀 구성부터 본 대회 참가까지 끊김 없이 경험할 수 있으며,
> SW마에스트로 프로그램의 연중 커뮤니티 활동과도 자연스럽게 이어집니다.

**1. DAKER 해커톤 페이지**

| 항목 | 내용 |
|------|------|
| **역할** | SW마에스트로 전용 해커톤 랜딩 페이지 — 대회 소개, 일정, 규칙, 과제, 리더보드, 수상작 아카이브를 한 곳에 통합 |
| **본 대회 연동** | 본 프롬프톤의 전체 참가자 여정(안내 → 참가 → 제출 → 결과 발표)이 이 페이지 하나로 완결 |
| **재활용성** | 대회 종료 후에도 연수생 포트폴리오·우수 사례 아카이브로 지속 활용 가능 |
| **브랜딩** | SW마에스트로 CI 적용 커스텀 배너·컬러·문구 지원 |

**2. 팀빌딩 페이지**

| 항목 | 내용 |
|------|------|
| **역할** | 개인 참가자가 팀원을 찾고 스스로 팀을 결성할 수 있는 매칭 페이지 |
| **기능** | 자기 소개·기술 스택·희망 역할 등록 → 참가자 간 상호 탐색 → 팀 초대·수락 → 자동 팀 등록 |
| **본 대회 연동** | 사전 모집 기간 중 팀 구성 완료 → 해커톤 당일 곧바로 실전 투입 가능 |
| **운영 이점** | 운영팀 수동 개입 없이 참가자 주도로 팀 결성이 완료되어 현장 혼선 최소화 |

**3. 그룹 결성 페이지**

| 항목 | 내용 |
|------|------|
| **역할** | 팀보다 상위 단위인 **그룹**(트랙·주제·소속·멘토링 조 등) 결성·관리 페이지 |
| **기능** | 그룹 생성·가입·탈퇴, 그룹별 대시보드, 그룹 단위 공지·커뮤니케이션 |
| **본 대회 연동** | SW마에스트로 멘토링 조·기수·트랙 단위 그룹을 미리 구성 → 그룹별 리더보드·분석 보고서 자동 생성 |
| **연중 확장성** | 본 프롬프톤 종료 후에도 그룹은 연수생 커뮤니티로 유지되어 후속 대회·학습 활동 기반으로 작동 |

**유기적 연동 효과**

| 단계 | 기존 방식 | DAKER 연동 방식 |
|------|----------|----------------|
| 참가 안내 | 이메일·공지 배포 | DAKER 해커톤 페이지 단일 링크 |
| 팀 구성 | 운영팀 수동 매칭·엑셀 관리 | **팀빌딩 페이지 자동 매칭** |
| 그룹 편성 | 별도 스프레드시트 | **그룹 결성 페이지 자동 집계** |
| 대회 참가 | 외부 시스템 이동 | DAKER 플랫폼 내 이탈 없이 완결 |
| 대회 후 | 결과 PDF 일회성 공유 | 아카이브·포트폴리오·그룹 활동 지속 |
