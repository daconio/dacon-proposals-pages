## CONTEXT — SW중심대학 경진대회란

과학기술정보통신부 주관, 전국 49개 SW중심대학이 참가하는 **국가 규모 AI·SW 경진대회**

### 대회 개요

| 항목 | 내용 |
|------|------|
| 주관 | 과학기술정보통신부 · SW중심대학협의회 |

| 평가 방식 | **알고리즘 경진대회 + AI서비스 개발(바이브코딩) 해커톤** 동시 진행 |
| 개최 주기 | 연 1회 (하반기) |
| 규모 | 전국 대학 **수천 명** 참가 |

### 2025년 본선 구조 (듀얼 트랙)

2025년 SW중심대학 경진대회 본선은 **2개 트랙을 동시에 운영**하는 구조로 진행되었습니다:

| 트랙 | 내용 | 평가 |
|------|------|------|
| **알고리즘 경진대회** | 데이터셋 기반 AI/ML 모델 경쟁 | 정확도 기반 자동 채점 |
| **AI서비스 해커톤** | AI 도구 활용 실제 서비스 개발 | 발표 심사 + 시연 |

→ 사전 연습 대회도 **본선과 동일한 듀얼 트랙 구조**로 운영해야 실전 대비 효과가 극대화됩니다.

### 왜 중요한가

- SW중심대학 **사업 평가에 직접 반영** (경진대회 참가·수상 실적)
- 학생 포트폴리오 및 **취업 연계** 핵심 경력
- 대학 브랜드 및 **AI 교육 역량** 대외 입증
- 과기정통부 **후속 사업 선정**에 영향

> 출처: 과기정통부 SW중심대학사업 운영 가이드라인 (2025), SW중심대학 경진대회 본선 요강 (2025)

## PROBLEM — 왜 사전 연습이 필요한가

본선 준비 부족은 곧 성과 지표 미달 — **실전 경험 없이 본선에 나가는 학생들**

### 현재 문제

- 본선이 **알고리즘 + AI서비스 해커톤 동시 진행**인데, 두 트랙을 모두 준비하는 학교가 드뭄
- 대부분의 학생이 **경진대회 경험 없이** 본선 참가
- 알고리즘 대회와 AI서비스 대회의 **평가 기준을 모르는 채** 출전
- 팀 빌딩, 발표, 데모 등 **실전 프로세스 미숙**
- 결과: 본선 입상률 저조 → SW중심대학 평가 지표 미달

### 타교와의 격차

| 구분 | 사전 대회 운영 학교 | 미운영 학교 |
|------|-------------------|-----------|
| 본선 참가 팀 수 | 5~10팀 이상 | 1~3팀 |
| 입상 실적 | 매년 입상 | 불규칙 |
| 학생 참여율 | 높음 (교내 분위기 형성) | 낮음 (소수 관심자만) |
| SW중심대학 평가 | 경진대회 항목 고득점 | 상대적 저평가 |

### 해결 방안

**사전 연습 대회를 통해:**
- 학생들이 대회 프로세스를 **사전 체험**
- 팀 빌딩 → 개발 → 발표 → 심사 **전 과정 경험**
- 본선과 동일한 환경에서 **실전 감각 축적**
- 우수 팀을 **조기 발굴·집중 육성**

### DACON — 알고리즘 경진대회

**AI 모델로 데이터를 분석한다**

- 데이터셋 기반 **AI/ML 모델 경쟁**

**대상:** **SW·AI 전공 학생** 중심 (컴퓨터공학과, 소프트웨어학과, 데이터사이언스)
