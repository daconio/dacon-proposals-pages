## 1. 왜 알고리즘 경진대회인가

### 배경

AI 전환이 전 산업으로 확산되면서, 기업·공공기관·대학·연구기관 모두 **AI 역량을 갖춘 인재를 발굴하고, 조직 내 AI 활용 역량을 실증할 수 있는 경진대회**의 필요성이 높아지고 있습니다.

알고리즘 경진대회는 실제 데이터를 기반으로 참가자들이 경쟁하며 최적의 솔루션을 도출하는 과정에서, **AI 인재를 발굴하고 조직의 기술 혁신 역량을 대내외에 입증**할 수 있는 가장 효율적인 방법입니다. 데이콘은 500개 이상의 대회 운영 경험과 자체 특허 기술을 바탕으로, 주최측의 목적에 최적화된 경진대회를 기획부터 성과 분석까지 올인원으로 운영 대행합니다.

### 3대 핵심 목표

| 목표 | 내용 | 기대 성과 |
|------|------|----------|
| **AI 인재 발굴** | 실전 데이터 기반 문제 해결 역량을 객관적으로 검증 | 우수 인재 풀 확보, 채용·육성 연계 |
| **기술 혁신 촉진** | 조직 내외부 데이터를 활용한 AI 모델 경쟁으로 혁신 솔루션 도출 | 실무 적용 가능한 AI 모델 확보 |
| **브랜드 제고** | 'AI 선도 조직'으로서의 대외 포지셔닝 | 조직 브랜드 강화, 생태계 리더십 확보 |

### 왜 지금인가

- **AI 전환 가속화**: 전 산업에서 AI 도입이 필수가 되면서, AI 역량 검증의 수요 급증
- **생성형 AI 확산**: ChatGPT 이후 AI 관심도 폭발, 경진대회 참여 의향 역대 최고
- **데이터 기반 의사결정**: 조직 내 축적된 데이터를 실전 문제에 활용할 인프라 필요
- **인재 경쟁 심화**: AI 인재 확보·육성을 위한 차별화된 프로그램 필요

> 데이콘은 8가지 대회 유형을 운영하며, 주최측의 목적·데이터·참가 대상에 따라 최적의 유형을 제안합니다.

### 유형별 상세

#### Type 1 — AI 알고리즘 대회

> 참가자가 본인의 코딩 역량과 AI를 활용하여 모델을 개발하고, 자동 평가 시스템으로 점수를 부여받으며, 실시간 리더보드를 통해 성과를 즉시 확인하는 가장 범용적인 AI 경진대회 유형

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 수요 예측, 이상 탐지, 분류/회귀, 최적화 |
| **과제 예시** | 제조 불량 예측, 금융 리스크 분류, 에너지 수요 예측, 고객 이탈 예측, 물류 최적화 |
| **방식** | 온라인 리더보드 기반 실시간 성능 경쟁 |
| **평가 기준** | 예측 정확도(정량 70%) + 코드 품질·설명가능성(정성 30%) |
| **결과물** | AI 모델 + 분석 보고서 → 주최측 성과 자료로 활용 |
| **대표 사례** | 토스 NEXT ML CHALLENGE |

#### Type 2 — AI 알고리즘 대회 on 코드 제출

> 참가자가 코드 자체를 제출하면 서버에서 자동 실행·평가하는 방식으로, 모델 재현성과 코드 품질까지 동시에 검증

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 비정형 데이터 AI (이미지, NLP, 음성), 대규모 모델 평가 |
| **과제 예시** | 제품 외관 검사, 의료 영상 분석, 문서 자동 분류, 감성 분석 |
| **방식** | 코드 제출 → 서버 자동 실행 → 리더보드 반영 |
| **평가 기준** | 모델 성능(정량 70%) + 코드 품질·혁신성(정성 30%) |
| **결과물** | AI 모델 + 실행 코드 + 기술 보고서 |
| **대표 사례** | 2025 Samsung AI Challenge |

#### Type 3 — 프롬프톤

> 플랫폼 내 탑재된 도구를 활용해 프롬프트를 직접 설계하고, 자동 평가 시스템을 통해 정량 점수를 부여받는 방식

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 프롬프트 엔지니어링, RAG 시스템 구축 |
| **과제 예시** | 도메인 특화 프롬프트 설계, 지식 검색 시스템 최적화 |
| **방식** | 플랫폼 내 프롬프트 작성 + 자동 평가 시스템 |
| **평가 기준** | 정량 자동 점수(70%) + 프롬프트 설계 품질(30%) |
| **결과물** | 프롬프트 세트 + 성능 보고서 |
| **대표 사례** | SW마에스트로 프롬프톤 |

#### Type 4 — AI 서비스 및 아이디어 공모전

> AI 기반 서비스·정책·아이디어를 기획·제안하는 공모전 방식으로, 기술 구현보다 기획력과 실용성을 중시

| 항목 | 내용 |
|------|------|
| **핵심 주제** | AI 서비스 기획, 정책 아이디어 제안, AI 활용 비즈니스 모델 |
| **과제 예시** | 공공 서비스 AI 활용 제안, 도메인 특화 AI 서비스 기획 |
| **방식** | 기획서·제안서 제출 + 발표 평가 |
| **평가 기준** | 실용성(40%) + 혁신성(30%) + 완성도(30%) |
| **결과물** | 기획서 + 발표 자료 |
| **대표 사례** | 수원시 정책 아이디어 제안 |

#### Type 5 — 온/오프 하이브리드 해커톤

> 온라인 예선과 오프라인 본선을 결합하여, 대규모 참가자 모집과 현장 집중 개발을 동시에 달성

| 항목 | 내용 |
|------|------|
| **핵심 주제** | AI 에이전트 개발, AI 기반 업무 자동화, AI 서비스 프로토타이핑 |
| **과제 예시** | 도메인 특화 챗봇 개발, AI 업무 자동화 도구, 생성형 AI 서비스 |
| **방식** | 온라인 예선(리더보드) → 오프라인 본선(해커톤 + 발표) |
| **평가 기준** | 완성도(40%) + 혁신성(30%) + 실용성(30%) |
| **결과물** | AI 애플리케이션 + 기술 보고서 |
| **대표 사례** | KT K intelligence 해커톤 |

#### Type 6 — 온/오프 하이브리드 해커톤 on 코드 제출

> 하이브리드 해커톤에서 코드 제출 기반 자동 평가를 적용하여 공정성과 재현성을 강화

| 항목 | 내용 |
|------|------|
| **방식** | 온라인 예선(코드 제출) → 오프라인 본선(해커톤 + 코드 제출 평가) |
| **평가 기준** | 모델 성능(정량) + 코드 품질 + 발표 평가 |
| **결과물** | 실행 코드 + AI 모델 + 발표 자료 |
| **대표 사례** | LG 오프라인 해커톤 |

#### Type 7 — 온라인 대회와 시상식

> 온라인으로 대회를 진행하고, 오프라인 시상식으로 마무리하여 참가자 접근성과 행사 격식을 모두 확보

| 항목 | 내용 |
|------|------|
| **방식** | 온라인 리더보드 경쟁 → 오프라인 시상식·네트워킹 |
| **평가 기준** | 정량 자동 평가 + 정성 심사 |
| **결과물** | AI 모델 + 분석 보고서 |
| **대표 사례** | 성균관대 Bias 발견 챌린지 |

#### Type 8 — 비공개 기업 해커톤

> 기업 내부 임직원 대상으로 비공개 운영하는 AI 해커톤으로, 조직 내 AI 역량 검증 및 문화 확산에 최적

| 항목 | 내용 |
|------|------|
| **참가 대상** | 주최 기업 임직원 한정 (인증 기반 접근 제한) |
| **방식** | 비공개 플랫폼 운영 → 내부 해커톤 → 내부 시상 |
| **평가 기준** | 주최측 맞춤 설계 |
| **결과물** | 내부 AI 솔루션 + 역량 분석 보고서 |

## 4. 대회 운영 프로세스

### 대회 운영 5단계

```
1단계: 상담 및 기획 → 2단계: 데이터 준비 및 검수 → 3단계: 플랫폼 구축 및 홍보 → 4단계: 대회 운영 및 평가 → 5단계: 결과 분석 및 후속 연계
```

#### 1단계: 상담 및 기획

| 활동 | 세부 내용 | 담당 |
|------|----------|------|
| **목표 설정** | 주최측과 사전 컨설팅을 통해 대회 목표·주제·일정·평가 방식 등 전반적인 운영 계획 수립 | 공동 |
| **대회 유형 선정** | 8가지 유형 중 주최측 목적에 최적화된 유형 제안 및 확정 | 공동 |

| **예산·일정 확정** | 범위·예산·일정 확정, 계약 체결 | 공동 |

## 5. 플랫폼 기능

> 데이콘은 두 개의 특화 플랫폼을 운영합니다. **DACON(dacon.io)**은 AI 알고리즘 경진대회 특화 플랫폼으로 리더보드, 자동 채점, 코드 제출 등 알고리즘 대회에 최적화된 기능을 제공합니다. **DAKER(daker.ai)**는 바이브코딩·서비스개발·프롬프트 특화 플랫폼으로 팀 협업, 실시간 공동 편집, 프롬프트 생성 도구 등을 제공합니다. 본 알고리즘 경진대회는 DACON 플랫폼을 기반으로 운영되며, 해커톤/프롬프톤 연계 시 DAKER를 함께 활용합니다.

### DACON 핵심 기능 (알고리즘 특화)

#### 온라인 코딩 플랫폼 (Jupyter LAB 환경)

- 실무 데이터와 온라인 실시간 코딩이 가능한 학습 환경 제공
- 별도 환경 설정 없이 브라우저에서 즉시 코드 작성·실행 가능
- GPU 포함 클라우드 컴퓨팅 환경 제공 (대회 규모에 따라 조정)

#### 베이스라인 코드 체계적 제공

- 참가자가 대회를 통해 '실전'을 학습할 수 있도록 체계적인 베이스라인 코드 제공
- EDA, 전처리, 모델링, 제출까지 전 과정을 포함한 튜토리얼 노트북
- 4만명이 경험한 AI 실무 교육 플랫폼

#### 과제 제출 · 채점 자동화

- 파일 업로드 기반 결과물 제출 (CSV, 모델 파일, 코드 등)
- **자동 채점 서버**: 제출 즉시 평가 지표 자동 산출 (RMSE, AUC, F1 등)
- 일일 제출 횟수 제한 설정 (과적합 방지)
- 제출 이력 관리 및 최고 점수 자동 갱신

#### 실시간 리더보드

- Public/Private 리더보드 이원화 (과적합 방지)
- 실시간 순위 변동 시각화
- 팀별/개인별 순위, 점수, 제출 횟수 표시
- 리더보드 동결(Freeze) 기능 (대회 종료 전 긴장감 유지)

#### 차세대 AI 리더보드 (LLM 모델 평가 시스템)

- **클라우드 기반 AI 모델 평가 체계**: GPU 클러스터를 활용한 자동 평가 시스템, 모델 크기·제공량에 따라 탄력적 평가
- **다양한 모델 제출 방식**: Docker 이미지, REST API, 파일 업로드 등 다양한 방식으로 모델 접수 및 평가
- **LLM 평가 프로토콜**: 문장이해, 추론, 도구 사용 능력, Agent 관련 능력 등 범용적 능력 평가 유형 정의
- **평가 지표 설계**: 일치율, 성공률, 효율성, 대응능력, 설명력 등 최소 5개 이상의 평가지표 자동화 (LLM-as-a-Judge 포함)
- **HuggingFace 연계**: 모델 등록, 평가 요청, 결과 표출 등 HuggingFace Space 연동
- **AI Hub 플랫폼 연계**: 평가 결과를 AI Hub에 표출, 모델 정보·평가결과·시각화 등 제공
- **평가 데이터 보안**: 비공개 평가 데이터를 활용한 신뢰성 있는 평가, 평가 시점에만 암호화 임시 스토리지에서 로드 후 즉시 삭제
- **실시간 평가 현황**: 평가 상태(대기 중, 평가 중, 완료 등) 실시간 표기, 지표별 점수·순위·평균 점수 등 시각화

#### 부정행위 탐지

- 특허 기반 자동 부정행위 탐지 시스템
- 코드 유사도 분석, 제출 패턴 이상 탐지
- 외부 데이터 사용 여부 자동 검사

### DAKER 핵심 기능 (바이브코딩·서비스개발·프롬프트 특화)
