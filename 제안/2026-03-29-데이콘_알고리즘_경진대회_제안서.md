## 1. 왜 알고리즘 경진대회인가

### 배경

단국대학교는 2024년 SW중심대학으로 선정되어 6년간 110억원을 지원받으며, 전교생 대상 SW·AI 필수 교양 교육을 실시하고 있습니다. 연간 3,000명 이상이 SW·AI 교과목을 이수하는 가운데, **교육 성과를 실증하고 학생들의 실전 역량을 검증할 수 있는 경진대회**의 필요성이 높아지고 있습니다.

알고리즘 경진대회는 SW중심대학의 **교육 성과 지표에 직접 기여**하는 가장 효율적인 방법입니다. 실제 데이터를 기반으로 단국대 SW·AI 전공 학생들이 경쟁하며 최적의 솔루션을 도출하고, 그 과정에서 학생들의 문제 해결 역량을 객관적으로 검증할 수 있습니다. 본 대회는 DAKER 바이브코딩 해커톤과 함께 **2개 대회 동시 개최 구조**로, SW중심대학 AI 교육의 종합적 성과를 대외에 입증합니다.

### 3대 핵심 목표

| 목표 | 내용 | 기대 성과 |
|------|------|----------|
| **SW중심대학 성과 실증** | 교육 이수 학생의 실전 문제 해결 역량 검증 | SW중심대학 성과 지표 달성, 정량 근거 확보 |
| **학생 역량 강화** | 실제 데이터 기반 AI 모델 경쟁으로 실전 경험 제공 | 수강생 대비 대회 참가율 향상, 포트폴리오 구축 |
| **교육 브랜드 제고** | 'AI 교육 선도 대학'으로서의 대외 포지셔닝 | 대학 홍보, SW중심대학 평가 우수 사례 |

### 왜 지금인가

- **SW중심대학 성과 입증 시기**: 2024년 선정 후 2년차, 교육 성과를 대내외에 입증해야 하는 적기
- **생성형 AI 확산**: ChatGPT 이후 AI 관심도 폭발, 학생들의 경진대회 참여 의향 역대 최고
- **전교생 SW·AI 교육 기반 확보**: 연간 3,000명+ 이수 학생을 실전 대회로 연결할 인프라 필요
- **2개 대회 패키지 시너지**: 알고리즘 대회 + 바이브코딩 해커톤 동시 개최로 다양한 역량 검증

## 4. 대회 운영 프로세스

> "단국대 SW중심대학 학생 맞춤형 온라인 경진대회 — 총 8주 운영"

### 전체 타임라인 (8주)

```
[W1] 기획/준비 → [W2] 홍보/모집(단국대 내부) → [W3~W6] 온라인 예선(4주) → [W7] 코드 검증 → [W8] 시상/결과 발표
```

### Phase 1: 기획 및 준비 (W1)

| 활동 | 세부 내용 | 담당 |
|------|----------|------|
| **대회 주제 확정** | 단국대 교육 과정 연계 과제 설계, 데이터 적합성 검토 | 공동 |
| **데이터 검증** | 데이터 품질 점검, 학습/평가 데이터 분리 | 데이콘 |
| **플랫폼 구축** | 대회 페이지 제작, 리더보드 세팅, 단국대 전용 접근 설정 | 데이콘 |
| **베이스라인 코드** | 참가자용 베이스라인 노트북 작성 (EDA + 모델 예시) | 데이콘 |
| **규정 수립** | 참가 규정, 데이터 사용 약관, 시상 규정 확정 | 공동 |

### Phase 2: 홍보 및 모집 — 단국대 내부 (W2)

| 활동 | 세부 내용 | 기대 효과 |
|------|----------|----------|
| **학과 공지** | 컴퓨터공학과, 소프트웨어학과, 데이터사이언스학과 등 공지 | 핵심 타겟 직접 도달 |
| **LMS / 학교 홈페이지** | 단국대 학습관리시스템 및 공식 홈페이지 배너 노출 | 전교생 인지 |
| **SW중심대학 사업단** | 사업단 공식 채널을 통한 홍보 | 사업단 참여 학생 유입 |
| **데이콘 타겟 홍보** | 단국대 재학생 대상 타겟 eDM 및 알림 | 데이콘 기가입 학생 도달 |
| **동아리·스터디** | AI/데이터 관련 교내 동아리 타겟 홍보 | 자발적 참여 유도 |

### Phase 3: 온라인 예선 (W3~W6, 4주)

| 활동 | 세부 내용 | 운영 방식 |
|------|----------|----------|
| **대회 진행** | 참가자 모델 제출, 실시간 리더보드 경쟁 | 24시간 자동 채점 |
| **Q&A 운영** | 커뮤니티 게시판 실시간 질의응답 | 데이콘 운영팀 상주 |
| **중간 공지** | 참가 현황, 리더보드 업데이트, 팁 공유 | 주간 뉴스레터 |
| **기술 지원** | 제출 오류, 환경 이슈 등 기술 문의 대응 | 실시간 CS |
| **부정행위 모니터링** | 코드 유사도 검사, 이상 패턴 탐지 | 자동 + 수동 병행 |

### Phase 4: 코드 검증 (W7)

| 활동 | 세부 내용 | 목적 |
|------|----------|------|
| **코드 재현성 검증** | 상위 팀 코드 실행, 결과 재현 확인 | 공정성 확보 |
| **부정행위 최종 점검** | 코드 유사도 분석, 외부 데이터 사용 여부 확인 | 규정 준수 확인 |
| **코드 리뷰** | 코드 품질, 문서화, 설명가능성 평가 | 정성 평가 반영 |
| **최종 순위 확정** | 정량+정성 종합 평가 후 최종 수상자 선발 | 최종 결과 확정 |

### Phase 5: 시상 및 결과 발표 (W8)

| 활동 | 세부 내용 | 비고 |
|------|----------|------|
| **결과 발표** | 최종 수상자 발표 (온라인 또는 교내 행사) | 단국대 채널 활용 |
| **시상** | 수상자 시상, 인증서 발급 | SW중심대학 사업단 공동 |
| **성과 보고서** | 대회 결과 리포트, SW중심대학 성과 지표 매핑 자료 작성 | 1주 내 완료 |
| **수상작 아카이빙** | 우수 코드 및 분석 보고서 교내 공유 | 교육 자료 재활용 |

## 5. DAKER 플랫폼 기능

> DAKER는 대회 기획부터 성과 분석까지 전 과정을 지원하는 올인원 AI 경진대회 플랫폼입니다.

### 대회 등록 · 참가 접수 시스템

- 대회 소개 페이지 자동 생성 (주제, 일정, 규정, 상금 정보)
- 개인정보처리동의서 전자 서명 수집
- 다단계 동의 체크 (데이터 활용, 초상권, 결과물 활용 등)
- 참가 현황 실시간 대시보드 (일별 신규 등록, 누적 참가자)

### 팀 구성 및 매칭 기능

- **원정대(Raid) 시스템**: 역할(기획/개발/디자인) 기반 자동 팀 매칭
- 팀 초대 링크 생성, 팀원 역할 관리
- 팀 내 실시간 채팅 및 파일 공유
- 팀 활동 로그 (기여도 분석 기초 자료)

### 과제 제출 · 채점 자동화

- 파일 업로드 기반 결과물 제출 (CSV, 모델 파일, 코드 등)

- 일일 제출 횟수 제한 설정 (과적합 방지)

### 커뮤니티 · 토론 게시판

- 운영진 공지 고정 기능
- 이미지/파일 첨부, 멘션(@), 좋아요 기능

### 작전실(War Room) 실시간 공동 편집

- 코드 공동 편집 및 버전 관리
