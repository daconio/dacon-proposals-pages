## 1. 왜 알고리즘 경진대회인가

### 배경

AI 전환이 전 산업으로 확산되면서, 기업·공공기관·대학·연구기관 모두 **AI 역량을 갖춘 인재를 발굴하고, 조직 내 AI 활용 역량을 실증할 수 있는 경진대회**의 필요성이 높아지고 있습니다.

알고리즘 경진대회는 실제 데이터를 기반으로 참가자들이 경쟁하며 최적의 솔루션을 도출하는 과정에서, **AI 인재를 발굴하고 조직의 기술 혁신 역량을 대내외에 입증**할 수 있는 가장 효율적인 방법입니다. 데이콘은 300회 이상의 대회 운영 경험과 자체 특허 기술을 바탕으로, 주최측의 목적에 최적화된 경진대회를 기획부터 성과 분석까지 올인원으로 운영 대행합니다.

### 3대 핵심 목표

| 목표 | 내용 | 기대 성과 |
|------|------|----------|
| **AI 인재 발굴** | 실전 데이터 기반 문제 해결 역량을 객관적으로 검증 | 우수 인재 풀 확보, 채용·육성 연계 |
| **기술 혁신 촉진** | 조직 내외부 데이터를 활용한 AI 모델 경쟁으로 혁신 솔루션 도출 | 실무 적용 가능한 AI 모델 확보 |
| **브랜드 제고** | 'AI 선도 조직'으로서의 대외 포지셔닝 | 조직 브랜드 강화, 생태계 리더십 확보 |

### 왜 지금인가

- **AI 전환 가속화**: 전 산업에서 AI 도입이 필수가 되면서, AI 역량 검증의 수요 급증
- **생성형 AI 확산**: ChatGPT 이후 AI 관심도 폭발, 경진대회 참여 의향 역대 최고
- **데이터 기반 의사결정**: 조직 내 축적된 데이터를 실전 문제에 활용할 인프라 필요
- **인재 경쟁 심화**: AI 인재 확보·육성을 위한 차별화된 프로그램 필요

## 2. 대회 유형 및 구성 옵션

### Type A — 알고리즘 (정형 데이터 예측)

> 참가자가 본인의 코딩 역량과 AI를 활용하여 코드를 제작하고, 자동 평가 시스템으로 점수를 부여받으며, 실시간 리더보드를 통해 성과를 즉시 확인하는 가장 범용적인 AI 경진대회 유형

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 수요 예측, 이상 탐지, 분류/회귀, 최적화 |
| **과제 예시** | 제조 불량 예측, 금융 리스크 분류, 에너지 수요 예측, 고객 이탈 예측, 물류 최적화 |
| **방식** | 온라인 리더보드 기반 실시간 성능 경쟁 |
| **평가 기준** | 예측 정확도(정량 70%) + 코드 품질·설명가능성(정성 30%) |
| **결과물** | AI 모델 + 분석 보고서 → 주최측 성과 자료로 활용 |

**진행 구조 (3단계):**

| 단계 | 내용 |
|------|------|
| **1단계** | 참가자 등록 |
| **2단계** | 예선 리더보드 및 코드 평가 |
| **3단계** | 본선 해커톤 / 발표 |

### Type B — 비정형 데이터 AI

> 이미지 분류, 객체 탐지, NLP 등 Vision/NLP 모델 경쟁

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 이미지 분류/탐지, 자연어 처리, 음성 인식, 문서 분석 |
| **과제 예시** | 제품 외관 검사, 의료 영상 분석, 문서 자동 분류, 감성 분석, 음성 인식 |
| **방식** | 온라인 리더보드 기반 실시간 성능 경쟁 |
| **평가 기준** | 모델 성능(정량 70%) + 코드 품질·혁신성(정성 30%) |
| **결과물** | AI 모델 + 기술 보고서 |

### Type C — 생성형 AI 활용

#### C-1: 프롬프트 엔지니어링

> 플랫폼 내 탑재된 툴을 활용해 프롬프트를 직접 생성하고, 자동 평가 시스템을 통해 정량 점수를 부여받는 방식

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 프롬프트 엔지니어링, RAG 시스템 구축 |
| **과제 예시** | 도메인 특화 프롬프트 설계, 지식 검색 시스템 최적화 |
| **방식** | 플랫폼 내 프롬프트 작성 + 자동 평가 시스템 |
| **평가 기준** | 정량 자동 점수(70%) + 프롬프트 설계 품질(30%) |
| **결과물** | 프롬프트 세트 + 성능 보고서 |

#### C-2: AI 서비스 개발 해커톤

> 참가자가 생성형 API를 활용하여 AI 서비스를 개발하는 해커톤 방식

| 항목 | 내용 |
|------|------|
| **핵심 주제** | AI 에이전트 개발, AI 기반 업무 자동화, AI 서비스 프로토타이핑 |
| **과제 예시** | 도메인 특화 챗봇 개발, AI 업무 자동화 도구, 생성형 AI 서비스 |
| **방식** | 결과물 제출 + 시연 평가 |
| **평가 기준** | 완성도(40%) + 혁신성(30%) + 실용성(30%) |
| **결과물** | AI 애플리케이션 + 기술 보고서 |

**진행 구조 (3단계):**

| 단계 | 내용 |
|------|------|
| **1단계** | 기획서 제출 (플랫폼 내 접수) |
| **2단계** | 참가자 확정 |
| **3단계** | 발표자료·산출물·시연자료 제출 |

### 대회 공개 범위 옵션

| 구분 | 공개(Public) | 비공개(Private) |

## 4. 대회 운영 프로세스

> "주최측 맞춤형 AI 경진대회 — 9단계 수행범위 + 표준 8주 운영 프레임워크"

### 수행범위 (설계 → 운영 → 검증)

#### 설계

| 단계 | 항목 | 세부 내용 |
|------|------|----------|
| **1** | 대회 데이터 구축 및 운영 계획 수립 | 주최측과 사전 컨설팅을 통해 대회 목표·주제·데이터·일정·평가 방식 등 전반적인 운영 계획 수립 |
| **2** | 문제(평가) 및 데이터 검토 | 과제 난이도 조정, 평가 지표 설계, 데이터 품질 점검 및 학습/평가 데이터 분리 |
| **3** | 대회/교육 설계 및 운영 | 대회 규정, 참가 요건, 시상 체계, 교육 연계 프로그램 등 상세 설계 |

#### 운영

| 단계 | 항목 | 세부 내용 |
|------|------|----------|
| **4** | 대회/교육 페이지 개설 및 운영 | DAKER 플랫폼에 대회 전용 페이지 구축, 참가 접수·공지·Q&A 운영 |
| **5** | 리더보드 운영 및 평가 | 실시간 자동 채점, Public/Private 리더보드 운영, 부정행위 모니터링 |
| **6** | 베이스라인 코드 제작 및 배포 | 참가자용 베이스라인 노트북 작성 (EDA + 모델 예시), 학습 자료 배포 |

#### 검증

| 단계 | 항목 | 세부 내용 |
|------|------|----------|
| **7** | 오프라인 경진대회 운영 및 평가 | 본선 해커톤·발표 운영, 현장 평가 진행 (해당 시) |
| **8** | 우승자 산출물 및 리포트 전달 | 상위 팀 코드 재현성 검증, 우수 모델·분석 보고서 취합 및 전달 |
| **9** | 최종 리포트 | 대회 결과 종합 보고서, 참가 통계, 성과 지표 매핑 자료 작성 |

### 전체 타임라인 (8주)

```
[W1] 기획/준비 → [W2] 홍보/모집 → [W3~W6] 온라인 예선(4주) → [W7] 코드 검증 → [W8] 시상/결과 발표
```

### Phase 1: 기획 및 준비 (W1)

| 활동 | 세부 내용 | 담당 |
|------|----------|------|
| **대회 주제 확정** | 주최측 목적에 맞는 과제 설계, 데이터 적합성 검토 | 공동 |
| **데이터 검증** | 데이터 품질 점검, 학습/평가 데이터 분리 | 데이콘 |
| **플랫폼 구축** | 대회 페이지 제작, 리더보드 세팅, 접근 권한 설정 | 데이콘 |
| **베이스라인 코드** | 참가자용 베이스라인 노트북 작성 (EDA + 모델 예시) | 데이콘 |
| **규정 수립** | 참가 규정, 데이터 사용 약관, 시상 규정 확정 | 공동 |

### Phase 2: 홍보 및 모집 (W2)

| 활동 | 세부 내용 | 기대 효과 |
|------|----------|----------|
| **주최측 내부 채널** | 주최측 공식 홈페이지, 사내/교내 공지, 이메일 등 | 핵심 타겟 직접 도달 |
| **데이콘 플랫폼 홍보** | 데이콘 메인 노출, 타겟 eDM, 푸시 알림 | 14만+ AI 인재 풀 도달 |
| **SNS·커뮤니티** | AI/데이터 관련 커뮤니티, SNS 채널 홍보 | 잠재 참가자 유입 |
| **언론·매체** | 보도자료 배포, AI 전문 매체 협력 (공개 대회 시) | 브랜드 노출 극대화 |
| **파트너 네트워크** | 관련 기관·단체·동아리 등 네트워크 활용 | 자발적 참여 유도 |

### Phase 3: 온라인 예선 (W3~W6, 4주)

| 활동 | 세부 내용 | 운영 방식 |
|------|----------|----------|
| **대회 진행** | 참가자 모델 제출, 실시간 리더보드 경쟁 | 24시간 자동 채점 |
| **Q&A 운영** | 커뮤니티 게시판 실시간 질의응답 | 데이콘 운영팀 상주 |
| **중간 공지** | 참가 현황, 리더보드 업데이트, 팁 공유 | 주간 뉴스레터 |
| **기술 지원** | 제출 오류, 환경 이슈 등 기술 문의 대응 | 실시간 CS |
| **부정행위 모니터링** | 코드 유사도 검사, 이상 패턴 탐지 | 자동 + 수동 병행 |

### Phase 4: 코드 검증 (W7)

| 활동 | 세부 내용 | 목적 |
|------|----------|------|
| **코드 재현성 검증** | 상위 팀 코드 실행, 결과 재현 확인 | 공정성 확보 |
| **부정행위 최종 점검** | 코드 유사도 분석, 외부 데이터 사용 여부 확인 | 규정 준수 확인 |
| **코드 리뷰** | 코드 품질, 문서화, 설명가능성 평가 | 정성 평가 반영 |
| **최종 순위 확정** | 정량+정성 종합 평가 후 최종 수상자 선발 | 최종 결과 확정 |

### Phase 5: 시상 및 결과 발표 (W8)

| 활동 | 세부 내용 | 비고 |
|------|----------|------|
| **결과 발표** | 최종 수상자 발표 (온라인 또는 오프라인 행사) | 주최측 채널 활용 |
| **시상** | 수상자 시상, 인증서 발급 | 주최측·데이콘 공동 |
| **성과 보고서** | 대회 결과 리포트, 주최측 성과 지표 매핑 자료 작성 | 1주 내 완료 |
| **수상작 아카이빙** | 우수 코드 및 분석 보고서 아카이빙 | 조직 내 재활용 가능 |

## 5. DAKER 플랫폼 기능

> DAKER는 대회 기획부터 성과 분석까지 전 과정을 지원하는 올인원 AI 경진대회 플랫폼입니다.

### 온라인 코딩 플랫폼 (Jupyter LAB 환경)

- 실무 데이터와 온라인 실시간 코딩이 가능한 학습 환경 제공
- 별도 환경 설정 없이 브라우저에서 즉시 코드 작성·실행 가능
- GPU 포함 클라우드 컴퓨팅 환경 제공 (대회 규모에 따라 조정)

### 베이스라인 코드 체계적 제공

- 참가자가 대회를 통해 '실전'을 학습할 수 있도록 체계적인 베이스라인 코드 제공
- EDA, 전처리, 모델링, 제출까지 전 과정을 포함한 튜토리얼 노트북
- 4만명이 경험한 AI 실무 교육 플랫폼

### 대회 등록 · 참가 접수 시스템

- 대회 소개 페이지 자동 생성 (주제, 일정, 규정, 상금 정보)
- 개인정보처리동의서 전자 서명 수집
- 다단계 동의 체크 (데이터 활용, 초상권, 결과물 활용 등)
- 참가 현황 실시간 대시보드 (일별 신규 등록, 누적 참가자)

### 팀 구성 및 매칭 기능

- **원정대(Raid) 시스템**: 역할(기획/개발/디자인) 기반 자동 팀 매칭
- 팀 초대 링크 생성, 팀원 역할 관리
- 팀 내 실시간 채팅 및 파일 공유
- 팀 활동 로그 (기여도 분석 기초 자료)

### 과제 제출 · 채점 자동화

- 파일 업로드 기반 결과물 제출 (CSV, 모델 파일, 코드 등)
- **자동 채점 서버**: 제출 즉시 평가 지표 자동 산출 (RMSE, AUC, F1 등)
- 일일 제출 횟수 제한 설정 (과적합 방지)
- 제출 이력 관리 및 최고 점수 자동 갱신

### 차세대 AI 리더보드 (LLM 모델 평가 시스템)

- **클라우드 기반 AI 모델 평가 체계**: GPU 클러스터를 활용한 자동 평가 시스템, 모델 크기·제공량에 따라 탄력적 평가
- **다양한 모델 제출 방식**: Docker 이미지, REST API, 파일 업로드 등 다양한 방식으로 모델 접수 및 평가
- **LLM 평가 프로토콜**: 문장이해, 추론, 도구 사용 능력, Agent 관련 능력 등 범용적 능력 평가 유형 정의
- **평가 지표 설계**: 일치율, 성공률, 효율성, 대응능력, 설명력 등 최소 5개 이상의 평가지표 자동화 (LLM-as-a-Judge 포함)
- **HuggingFace 연계**: 모델 등록, 평가 요청, 결과 표출 등 HuggingFace Space 연동
- **AI Hub 플랫폼 연계**: 평가 결과를 AI Hub에 표출, 모델 정보·평가결과·시각화 등 제공
- **평가 데이터 보안**: 비공개 평가 데이터를 활용한 신뢰성 있는 평가, 평가 시점에만 암호화 임시 스토리지에서 로드 후 즉시 삭제
- **실시간 평가 현황**: 평가 상태(대기 중, 평가 중, 완료 등) 실시간 표기, 지표별 점수·순위·평균 점수 등 시각화

### 실시간 리더보드

- Public/Private 리더보드 이원화 (과적합 방지)
- 실시간 순위 변동 시각화
- 팀별/개인별 순위, 점수, 제출 횟수 표시
- 리더보드 동결(Freeze) 기능 (대회 종료 전 긴장감 유지)

### 커뮤니티 · 토론 게시판

- 대회 전용 Q&A 게시판 (카테고리별 분류)
- 코드 공유·토론 게시판 (마크다운 + 코드 하이라이팅)
- 운영진 공지 고정 기능
- 이미지/파일 첨부, 멘션(@), 좋아요 기능

### 작전실(War Room) 실시간 공동 편집

- 팀 전용 실시간 협업 공간
- 코드 공동 편집 및 버전 관리
- 실험 로그 공유 (하이퍼파라미터, 성능 기록)
- 화이트보드 기능 (아이디어 브레인스토밍)

### GitHub 연동

- GitHub 저장소 자동 연결
- 커밋 이력 기반 기여도 자동 산출
- 코드 리뷰 및 이슈 트래킹 연계
- 최종 제출물 GitHub 자동 아카이빙

### 업적 시스템 · 배지

- 대회 참가, 수상, 커뮤니티 활동 기반 디지털 배지 발급
- 스킬 레벨 시각화 (데이터 분석, ML, DL, NLP 등 영역별)
- 프로필 페이지 업적 표시
- 취업·이직·승진 시 역량 증빙 자료로 활용

### 무손실 데이터 저장소 (Archive)

- 대회 전 과정 데이터 아카이빙 (제출물, 코드, 토론, 리더보드)
- 주최측 전용 관리자 대시보드
- 대회 종료 후에도 데이터 영구 보존
- 결과 리포트 자동 생성 (참가 통계, 성과 분석)

### DASCHOOL 교육 연계

- 학습과학 기반 교재와 서비스 제공
- 분산 학습 및 분산 연습 교재 구성
- 과정 평가와 결과 평가 트랙 운영
- 대회와 교육을 연계한 일체형 AI 역량 강화 프로그램

### 민간자격증 제공

- **Operator**: AI 도구 활용 기본 역량 인증
- **Analyst**: 데이터 분석 및 모델링 역량 인증
- **Master**: 고급 AI 솔루션 설계 역량 인증

## 6. 오프라인 경진대회 현장 운영

> 데이콘은 온/오프라인 경진대회 동시 운영 역량을 보유한 유일한 기업입니다. 예선과 본선을 유기적으로 연계하여 완성도 높은 경진대회를 기획하며, 참가자 경험을 중심에 둔 설계와 검증된 운영 역량을 제공합니다.

### 현장 운영

| 항목 | 세부 내용 |
|------|----------|
| **행사장 당일 준비** | 장소 셋업, 네트워크 환경 점검, 좌석 배치, 안내 사인물 설치 |
| **사회자/진행요원** | 전문 사회자 및 진행요원 배치, 진행 슬라이드 제작·운영 |
| **실시간 피드백** | 참가자 대상 실시간 피드백 제공 및 기능 테스트 독려 |
| **참여자 식사/다과** | 식사·다과·간식 제공 (야간 해커톤 포함) |
| **안전 운영** | 야간 포함 안전한 운영 환경 보장 |

### 산출물 취합 및 심사 운영

| 항목 | 세부 내용 |
|------|----------|
| **산출물 접수** | 각 트랙별 산출물을 데이콘 플랫폼으로 접수·관리 |
| **심사 장소 구성** | 심사 장소 배치 및 장비 셋업, 심사 운영 |
| **프로젝트 발표** | 심사위원 대상 프로젝트 발표 진행 |
| **질의응답 및 기술 검증** | 심사위원과 참가자 간 질의응답, 기술적 검증 수행 |

### 심사 및 시상식 진행

## 8. 기대 효과 (KPI)

### 3대 기대 효과

#### Talent (인재 발굴)

- 대회를 통해 **AI 역량이 검증된 우수 인재**를 객관적 지표로 발굴
- 상위 수상자의 접근 방법, 코드 품질이 조직의 인재 평가 기준으로 활용
- DAKER 스킬 리포트를 통한 참가자별 역량 정량 분석 → 채용·육성 의사결정 지원

#### Brand (브랜드)

- 대회 개최 자체가 **'AI를 진지하게 추구하는 조직'** 이라는 강력한 시그널
- 데이콘 커뮤니티 및 AI 전문 매체를 통한 자연스러운 브랜드 노출
- AI 인재 시장에서의 **조직 인지도 및 선호도 제고**

#### Innovation (기술 혁신)

- 다양한 참가자의 접근법을 통해 **기존에 시도하지 못한 혁신적 솔루션** 발굴
- 대회 데이터셋과 우수 모델은 조직 내부 R&D 자료로 재활용
- 대회 수상작의 실무 적용을 통한 실질적 비즈니스 임팩트 창출
