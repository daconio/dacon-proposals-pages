Chapter 01

DAKER 플랫폼 매뉴얼

데이터 경진대회 및 해커톤 관리를 위한 종합 플랫폼 daker.ai의 공식 사용자 매뉴얼입니다.

1. 개요

1.1 DAKER란?

DAKER는 데이터 사이언스 커뮤니티를 위한 종합 플랫폼이다. DACON(데이콘)과 연동되어, 경진대회 참가자들이 팀을 구성하고 해커톤에 참여하며, 커뮤니티에서 지식을 공유할 수 있는 환경을 제공한다.

1.2 DACON 생태계 내 위치

플랫폼주요 영역특징
DACON (dacon.io)알고리즘 경진대회데이터 사이언스, ML/DL 모델 성능 경쟁
DAKER (daker.ai)바이브코딩 / 서비스 개발 / 프롬프트실무 중심 해커톤, 팀 빌딩, 학습

1.3 사용자 역할

User (일반 사용자)

해커톤 참가, Raid 팀 활동, 학습, 커뮤니티 참여

/expert/*
Admin (관리자)

플랫폼 전체 관리, 해커톤 생성, 사용자 관리

3. Hackathons (해커톤 관리)

3.1 해커톤 분류

Premium

데이콘 직접 운영, 상금 있음

Standard

주최측 운영, 상금 있음

Community

커뮤니티 운영, 상금 없음

Private

비공개 운영, 상금 선택

3.2 해커톤 생명주기

Hackathon Lifecycle - State Flow
draft초안 작성
→
ready게시 대기
→
published참가 접수 중
→
evaluation심사 진행
→
closed결과 발표

3.3 스테이지 시스템

스테이지 유형설명
online_prelim온라인 예선 (기획서 제출, 온라인 투표)
offline_hackathon오프라인 해커톤 (현장 개발)
offline_eval오프라인 평가 (현장 투표/심사)
final_presentation최종 발표 (발표자료 제출)

3.4 참가 및 팀 빌딩

  • 해커톤 상세 페이지에서 참가 신청
  • 개인 참가 또는 팀 참가 선택
  • 팀원 초대 (이메일/링크)
  • Raid Group 연계 자동 팀 구성
  • 참가 조건 확인 (최소/최대 인원)

3.5 심사 시스템

구조화 채점

항목별 점수 입력 + 코멘트 작성. 섹션별 최대 점수 설정 가능.

Excel 채점

Excel 파일 업로드 후 자유 양식으로 평가. 점수/코멘트/체크박스 열 지원.

ELO 1v1 아레나

두 제출물 비교 승패 결정. ELO 레이팅 알고리즘 자동 순위 산출.

ELO 가중치 설정

평가자 가중치(기본 70%)와 참가자 가중치(기본 30%)를 별도 설정할 수 있으며, 두 점수의 가중 합산으로 최종 순위를 결정한다.

5. Community (커뮤니티)

5.1 게시판 시스템

/community 경로에서 접근하며, 카테고리별 게시판 시스템을 제공한다. Tiptap 리치 텍스트 에디터 또는 Markdown을 사용하여 포스트를 작성한다.

5.2 핵심 기능

포스트 CRUD

리치 텍스트 작성, 조회수 카운트, 소프트 삭제 처리

댓글 & 투표

대댓글 2단계, 업보트/다운보트, 실시간 집계

태그 시스템

다중 태그, 태그별 필터링, 인기 태그

라이브러리

유용한 리소스 모음 공간 (/community/library)

성능 최적화를 위해 communityCache.ts 캐시 시스템이 적용되어 있다.

6.3 게이미피케이션

Activity Rings

학습 시간 / 퀴즈 완료 / 연속 학습 3개의 링. 링 완성 시 보너스 포인트 지급.

배지 시스템

streak, learning, quiz, team, special 5개 카테고리. 조건 달성 시 자동 부여.

학습 포인트 (LP)

링 완성, 배지 획득, 팀 챌린지 등으로 획득. 힌트 구매 등에 소비.

6.4 팀 학습

Raid(팀) 단위의 팀 일일 통계, 팀 챌린지, 팀 스코어 경쟁, 관리자용 팀 학습 대시보드를 지원한다.

6.5 학습 트랙 & 수강 배정

여러 교재를 묶어 학습 경로를 구성하는 트랙 기능과, 관리자가 사용자에게 특정 교재를 배정하는 수강 배정 기능을 제공한다.

11. 디자인 시스템: The Editorial Architect

  • 채움 단계 시스템: 색상이 아닌 폰트/테두리로 구분 (색각 장애 접근성)

12.4 활동 등급 (Activity Class A/B/C)

Class A — 성과형

대회 제출, 수상, 상위 10%, 코드 베스트, 프로젝트 완료, 코스 완료

Class B — 기여형

코드 공유, 포스트 작성, 커밋, PR, 답변 채택, 원정대 참여/생성

Class C — 참여형

출석, 댓글 조회, 투표, 일일 채팅, 보이스 참여, 스테이지 완료

12.7 휴면 페널티 (Decay)

17. 데이터 분석 & 로드맵

DAKER 플랫폼의 데이터 분석 현황과 강화 계획을 정리한다.

17.1 현재 데이터 분석 기능

Admin Dashboard
  • 플랫폼 통계 (DAU, MAU, 가입자)
  • 해커톤별 참가율/제출율
  • 사용자 활동 트렌드
  • 커뮤니티 게시물 통계
Banner Analytics
  • 배너별 노출수/클릭수
  • CTR(클릭률) 추적
  • 시간대별 성과 분석
  • 배너 A/B 비교 기초 데이터
Survey Analytics
  • 응답률 추적
  • 질문별 분포 분석
  • 응답 시간 분석
  • AI 기반 응답 요약

17.2 데이터 분석 강화 로드맵

Phase 1: A/B 테스트 프레임워크 (Q2 2026)

  • 배너 A/B 테스트: 동일 슬롯에 2개 배너 노출, CTR 비교 자동화
  • 해커톤 랜딩 A/B: 설명 문구, CTA 버튼, 레이아웃 변형 테스트
  • 이메일 제목 A/B: 발송 그룹 분할, 오픈율/클릭률 비교
  • 트래픽 분할 비율 설정 (50/50, 70/30 등)
  • 통계적 유의성 자동 판정 (p-value < 0.05)

Phase 2: 사용자 행동 분석 (Q3 2026)

  • Funnel 분석: 가입 → 해커톤 참가 → 제출 → 수상 각 단계 이탈률
  • Cohort 분석: 월별 가입 코호트의 리텐션율 추적
  • 학습 경로 분석: 코스 완료율, 학습 → 해커톤 전환율
  • 팀 성과 분석: AI 매칭 팀 vs 자유 팀 성과 비교

Phase 3: AI 기반 예측 분석 (Q4 2026)

  • 이탈 예측: 학습 중단 위험 사용자 조기 감지 & 개입
  • 성과 예측: 학습 이력 기반 해커톤 수상 확률 예측
  • 콘텐츠 추천: 사용자 행동 패턴 기반 학습 콘텐츠 개인화
  • 최적 팀 크기 분석: 과거 데이터 기반 해커톤 유형별 적정 팀원 수 도출

Phase 4: 실시간 대시보드 (Q1 2027)

  • 해커톤 실시간 모니터링: 참가자 활동, 제출 현황, 서버 부하
  • 학습 실시간 추적: 강의 시청률, 퀴즈 정답률 실시간 표시
  • KPI 자동 리포팅: 주간/월간 성과 리포트 자동 생성 (Gemini)
  • 이상 탐지: 비정상 트래픽, 어뷰징 패턴 자동 알림
