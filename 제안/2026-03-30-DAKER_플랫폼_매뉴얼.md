## 1. 개요

### 1.1 DAKER란?

DAKER는 데이터 경진대회 및 해커톤 관리를 위한 풀스택 웹 애플리케이션이다. DACON(데이콘) 생태계와 연동되어, 경진대회 참가자들이 팀을 구성하고 해커톤에 참여하며, 커뮤니티에서 지식을 공유할 수 있는 종합 플랫폼을 제공한다.

- **운영 URL**: https://daker.ai
- **테스트 환경**: https://test1.daker.ai
- **개발 환경**: http://localhost:5000

### 1.2 DACON 생태계 내 위치

| 플랫폼 | 주요 영역 | 특징 |
|--------|----------|------|
| **DACON** (dacon.io) | 알고리즘 경진대회 | 데이터 사이언스, ML/DL 모델 성능 경쟁 |
| **DAKER** (daker.ai) | 바이브코딩 / 서비스 개발 / 프롬프트 | 실무 중심 해커톤, 팀 빌딩, 학습 |

DAKER는 DACON SSO를 통해 사용자 인증을 공유하며, DACON의 프로필 정보를 연동한다.

### 1.3 사용자 역할

DAKER 플랫폼은 3가지 핵심 사용자 역할을 지원한다.

| 역할 | 접근 경로 | 설명 |
|------|----------|------|
| **User** (일반 사용자) | `/public/*` | 해커톤 참가, Raid 팀 활동, 학습, 커뮤니티 참여 |
| **Expert** (심사위원) | `/expert/*` | 채점표 작성, 해커톤 평가, ELO 아레나 심사 |
| **Admin** (관리자) | `/admin/*` | 플랫폼 전체 관리, 해커톤 생성, 사용자/콘텐츠 관리 |

각 역할은 전용 레이아웃을 통해 독립적인 화면 구성을 가지며, `ProtectedRoute` 컴포넌트로 접근 권한이 제어된다.

### 2.2 인증 구조

DAKER는 다중 인증 전략을 지원하며, DACON SSO가 기본 인증 방식이다.

```
사용자 브라우저
     │
     ├── DACON SSO (주 인증) ──→ /api/auth/dacon/login
     │                             │
     │                             ▼
     │                       DACON OAuth Server
     │                             │
     │                             ▼
     │                       /api/auth/dacon/callback
     │
     ├── Google OAuth ──────→ /api/auth/google
     ├── GitHub OAuth ──────→ /api/auth/github
     └── Discord OAuth ─────→ /api/auth/discord
                                   │
                                   ▼
                            Passport.js (세션 관리)
                                   │
                                   ▼
                            PostgreSQL Session Store
```

**역할 기반 접근 제어(RBAC)**:
- `user`: 일반 사용자 (DACON SSO 로그인)
- `expert`: 심사위원 (별도 Expert 로그인)
- `admin`: 관리자 (Admin 전용 로그인)

```
App.tsx (메인 라우터)

  │     ├── PublicHeader (네비게이션 바)
  │     └── 페이지 컴포넌트
  │
  ├── AdminLayout (/admin/*)
  │     ├── 사이드바 네비게이션

  │
  └── ExpertLayout (/expert/*)

```

## 3. Hackathons (해커톤 관리)

### 3.1 해커톤 개념

DAKER의 해커톤은 온/오프라인 경진대회를 통합 관리하는 시스템이다. 프롬프트 해커톤, 개발 해커톤, 분석 해커톤, 아이디어 해커톤 등 다양한 템플릿을 지원한다.

### 3.2 해커톤 분류

| 유형 | 설명 | 상금 | 운영 주체 |
|------|------|------|----------|
| **Premium** | 데이콘 직접 운영 | 있음 | DACON |
| **Standard** | 주최측 직접 운영 | 있음 | 주최측 |
| **Community** | 커뮤니티 해커톤 | 없음 | 주최측 |
| **Private** | 비공개 해커톤 | 선택 | 데이콘/주최측 |

### 3.3 해커톤 생명주기

```
draft ──→ ready ──→ published ──→ evaluation ──→ closed
  │                    │              │            │
  │ 관리자 작성       │ 공개        │ 제출 마감   │ 결과 발표
  │ 설정 완료         │ 참가 접수    │ 심사 진행   │ 완료
  └────────────────────┴──────────────┴────────────┘
```

**상태 설명**:
- `draft`: 초안 상태. 관리자만 볼 수 있으며, 기본 정보와 설정을 입력하는 단계.
- `ready`: 게시 대기. 모든 설정이 완료되어 게시 가능한 상태.
- `published`: 진행 중. 참가자에게 공개되며, 등록 접수 및 제출물 접수가 진행됨.
- `evaluation`: 심사 중. 제출이 마감되고 심사위원이 평가를 진행하는 단계.
- `closed`: 종료. 결과 발표가 완료된 최종 상태.

### 3.4 해커톤 스테이지(단계) 시스템

하나의 해커톤은 여러 스테이지로 구성될 수 있다.

| 스테이지 유형 | 설명 |
|-------------|------|
| `online_prelim` | 온라인 예선 (기획서 제출, 온라인 투표) |
| `offline_hackathon` | 오프라인 해커톤 (현장 개발) |
| `offline_eval` | 오프라인 평가 (현장 투표/심사) |
| `final_presentation` | 최종 발표 (발표자료 제출) |

### 3.5 해커톤 생성 (Admin)

관리자는 `/admin/hackathons/new` 경로에서 새 해커톤을 생성한다.

**필수 입력 항목**:
- 기본 정보: 제목, 태그라인, 상세 설명 (Markdown)
- 디자인: 썸네일, 헤더 이미지, 테마 색상
- 참가 조건: 최소/최대 팀 인원, 최대 참가팀 수
- 일정: 등록 기간, 대회 기간
- 평가 기준: 평가 가중치 (평가자/참가자 비율)
- 주최자 정보: 주최측 이름, 연락처, 로고

**선택 입력 항목**:
- 외부 링크 (웹사이트, Discord)
- 상금 정보
- 규칙/가이드라인
- 동의 사항
- Raid Group 연결

### 3.6 참가 신청 및 팀 빌딩

1. 사용자가 해커톤 상세 페이지(`/devthon/:slug`)에서 참가 신청
2. 팀 구성: 개인 참가 또는 팀 참가 선택
3. 팀원 초대: 이메일 또는 링크 초대
4. Raid Group을 통한 팀 자동 구성 지원

### 3.7 제출물 관리

참가자는 해커톤 기간 중 제출물을 등록하고 수정할 수 있다.

- 제출물 형태: 프로젝트 설명, 발표 자료, 소스코드 링크, 데모 URL
- 제출 상태 추적: 미제출 → 임시저장 → 최종 제출
- 제출 마감 시간 관리

### 3.8 심사 시스템 (Expert Portal)

심사위원은 `/expert/*` 경로의 전용 포털에서 평가를 진행한다.

**평가 방식**:
1. **구조화된 채점**: 항목별 점수 + 코멘트
2. **Excel 기반 채점**: 자유 양식 Excel 업로드
3. **ELO 1v1 아레나**: 두 제출물을 비교하여 승패를 결정하는 방식

**ELO 아레나 평가**:
- 참가자 또는 심사위원이 두 제출물을 비교
- 승/패/무승부 선택
- ELO 레이팅 알고리즘으로 순위 자동 산출
- 평가자 가중치와 참가자 가중치를 별도로 설정 가능 (기본: 평가자 70%, 참가자 30%)

### 3.9 리더보드

해커톤별 리더보드는 실시간으로 순위를 표시한다.

- 스테이지별 점수 집계
- ELO 레이팅 기반 순위
- 심사위원 + 참가자 투표 가중 합산
- 수상 범위 설정 (상위 N위까지 수상)
- 수상 명칭 커스터마이징 (대상, 최우수상, 우수상 등)

## 4. Raids / Basecamp (팀 경쟁)

### 4.1 Raid 개념

Raid는 DAKER의 팀 기반 활동 단위이다. 해커톤 참가, 학습, 또는 프로젝트 수행을 위해 팀원을 모집하고 협업하는 공간을 제공한다.

### 4.2 Raid Group (그룹)

Raid Group은 여러 Raid(팀)를 묶는 상위 단위이다. 하나의 해커톤이나 교육 프로그램에 속하는 모든 팀을 관리한다.

**참여 유형**:
| 유형 | 설명 |
|------|------|
| `hackathon_only` | 해커톤 전용 |
| `learning_only` | 학습 전용 |
| `hackathon_and_learning` | 해커톤 + 학습 통합 |

**가입 방식**:
| 방식 | 설명 |
|------|------|
| `open` | 누구나 자유 가입 |
| `invite_only` | 초대를 받은 사용자만 가입 |
| `admin_only` | 관리자가 직접 추가 |

### 4.3 Raid(팀) 구조

각 Raid는 하나의 팀을 나타내며 다음 정보를 포함한다.

- 팀 이름, 설명, 분야 태그
- 리더 및 멤버 목록
- 모집 상태 (모집 중 / 모집 완료)
- 역할 배정 (리더, 멤버 등)
- 해커톤 연결 (선택)

### 4.4 멤버 관리

**멤버 역할**:
| 역할 | 권한 |
|------|------|
| `owner` | 그룹 전체 관리, 삭제 권한 |
| `admin` | 멤버 관리, 설정 변경 |
| `leader` | 팀 리더, 제출물 관리 |
| `member` | 일반 팀원 |

**멤버 추가 방법**:
1. 이메일 초대 (등록되지 않은 사용자도 사전 등록 가능)
2. 초대 링크 공유 (토큰 기반, 유효기간/사용횟수 제한)
3. 관리자 직접 추가
4. 사전 등록 멤버 자동 활성화 (DACON SSO 로그인 시)

### 4.5 War Room (실시간 협업)

War Room은 Yjs 기반의 실시간 협업 편집 공간이다.

- WebSocket을 통한 실시간 동시 편집
- Tiptap 리치 텍스트 에디터 사용
- 팀원 간 실시간 커서 표시
- 문서 히스토리 관리

### 4.6 팀 내 포스트 및 토론

각 Raid 내에서 팀원끼리 포스트를 작성하고 토론할 수 있다.

- Raid 포스트 CRUD (`/public/raids/:id/posts`)
- 포스트 카테고리 분류
- 댓글 및 대댓글

## 5. Community (커뮤니티)

### 5.1 게시판 시스템

커뮤니티는 `/community` 경로에서 접근하며, 카테고리별 게시판 시스템을 제공한다.

### 5.2 포스트 CRUD

- **작성**: 리치 텍스트 에디터 (Tiptap) 또는 Markdown 지원
- **읽기**: 조회수 카운트, 관련 포스트 추천
- **수정**: 작성자 본인만 수정 가능
- **삭제**: 소프트 삭제 처리

### 5.3 댓글 및 투표

- 댓글 및 대댓글 (2단계)
- 업보트/다운보트 시스템
- 댓글 수, 투표 수 실시간 표시
- 캐시 시스템을 통한 성능 최적화 (`communityCache.ts`)

### 5.4 태그 시스템

- 포스트에 다중 태그 부여 가능
- 태그별 필터링 및 검색
- 인기 태그 목록

### 5.5 라이브러리

커뮤니티 라이브러리(`/community/library`)는 유용한 리소스를 모아놓은 공간이다.

## 6. Learning (학습)

### 6.1 학습 콘텐츠 구조

DAKER의 학습 시스템은 교재(Material/Course) 기반으로 구성된다.

```
학습 카테고리 (Learning Category)
  └── 교재 (Material/Course)
        ├── 스테이지 1: 콘텐츠 (content)
        ├── 스테이지 2: 퀴즈 (quiz)
        ├── 스테이지 3: 프롬프트 (prompt)
        ├── 스테이지 4: PDF (pdf)
        └── 스테이지 5: 플래시카드 (flashcard)
```

**교재 상태**: `draft` → `published` → `archived`

**교재 난이도**: `beginner` | `intermediate` | `advanced`

### 6.2 스테이지 유형

| 유형 | 설명 |
|------|------|
| `content` | 마크다운 텍스트 콘텐츠 |
| `quiz` | 객관식/주관식 퀴즈 |
| `prompt` | AI 프롬프트 실습 |
| `pdf` | PDF 문서 뷰어 |
| `flashcard` | 플래시카드 암기 학습 |

### 6.3 학습 진도 관리

사용자별 학습 진도를 추적한다.

- 진도 상태: `not_started` → `in_progress` → `completed`
- 완료된 스테이지 목록 관리
- 총 학습 시간 기록 (초 단위)
- 스테이지별 학습 시간 맵
- 학습 완료 횟수 (반복 학습 지원)
- 최근 접근 시점 기록

### 6.4 플래시카드

플래시카드는 핵심 개념을 반복 학습할 수 있는 카드 형태의 학습 도구이다.

- 앞면/뒷면 카드 형태
- 스테이지 내에 포함
- 학습자가 직접 플래시카드 세트 생성 가능

### 6.5 배지 & 링 (Gamification)

학습 시스템은 게이미피케이션 요소를 포함한다.

**Activity Rings (일일 목표)**:
- 학습 시간 링 (목표 분)
- 퀴즈 완료 링 (목표 개수)
- 연속 학습 링 (목표 일수)
- 링 완성 시 보너스 포인트 지급

**배지 시스템**:
| 카테고리 | 예시 |
|---------|------|
| `streak` | 연속 학습 배지 |
| `learning` | 학습 시간 달성 배지 |
| `quiz` | 퀴즈 정답률 배지 |
| `team` | 팀 활동 배지 |
| `special` | 특별 이벤트 배지 |

**학습 포인트(LP)**:
- 링 완성, 배지 획득, 팀 챌린지 완료 등을 통해 획득
- 힌트 구매 등에 소비 가능

### 6.6 팀 학습

Raid(팀) 단위의 학습 게이미피케이션을 지원한다.

- 팀 일일 통계 (학습 시간, 퀴즈 수, 목표 달성 인원)
- 팀 챌린지 (주간 학습 목표, 팀 연속 달성)
- 팀 스코어 및 팀간 경쟁
- 관리자용 팀 학습 대시보드

### 6.7 학습 트랙

학습 트랙은 여러 교재를 묶어 하나의 학습 경로를 구성하는 기능이다.

- 사용자가 직접 트랙 생성 가능
- 교재 순서 지정
- 진도 추적

### 6.8 수강 배정

관리자가 사용자에게 특정 교재를 배정하는 기능이다.

- 개별 또는 그룹 단위 배정
- 배정 취소 기능
- 배정된 학습 현황 대시보드

## 8. Surveys (설문)

### 8.1 설문 개요

DAKER는 강력한 설문 시스템을 내장하고 있으며, 이벤트/해커톤과 연계된 설문 및 독립 설문을 지원한다.

### 8.2 설문 빌더

관리자는 `/admin/surveys/new`에서 설문을 생성하고, `/admin/surveys/:id/builder`에서 질문을 편집한다.

**설문 설정 항목**:
- 제목, 설명, 템플릿 유형
- 대상 역할 (participant, expert, learner 등)
- 익명 여부
- 복수 응답 허용
- 예상 소요 시간
- 인센티브 (포인트, 추첨 응모)
- 배너 설정 (이미지 또는 텍스트)
- 동의 사항

### 8.3 질문 유형

| 유형 | 설명 |
|------|------|
| `single_choice` | 단일 선택 (라디오 버튼) |
| `multiple_choice` | 복수 선택 (체크박스) |
| `short_text` | 단답형 |
| `long_text` | 서술형 |
| `rating` | 평점 (별점) |
| `ranking` | 순위 매기기 |
| `nps` | NPS (Net Promoter Score) |
| `email` | 이메일 입력 |
| `number` | 숫자 입력 |

**고급 기능**:
- 조건부 로직: 이전 답변에 따라 질문 표시/숨김
- 첨부 파일 지원
- 기타 답변 허용 (`allowOther`)

### 8.4 설문 배포

설문은 다양한 채널로 배포할 수 있다.

- 이메일 발송
- 알림 (인앱 알림)
- 웹 푸시
- 리마인더 자동 발송 설정

### 8.5 응답 분석

`/admin/surveys/:id/analytics`에서 설문 결과를 분석한다.

**분석 지표**:
- 총 응답 수, 완료율, 평균 응답 시간
- 일별 응답 추이 차트
- 디바이스별 분포 (데스크탑/모바일/태블릿)
- NPS 점수
- 질문별 통계 (평균, 중앙값, 분포, 워드 클라우드)

## 9. Scoring (채점)

### 9.1 채점 시스템 개요

DAKER는 이벤트/해커톤의 심사를 위한 다양한 채점 방식을 제공한다.

### 9.2 채점표 유형

| 유형 | 설명 |
|------|------|
| **Manual (새채점표)** | 섹션별 점수와 코멘트를 구조화하여 입력 |
| **AI 채점표** | AI가 생성한 채점 기준을 기반으로 평가 |
| **Excel 채점표** | Excel 파일 업로드 후 자유 양식으로 평가 |

### 9.3 Excel 기반 채점

1. 관리자가 Excel 채점표를 업로드 (채점 항목 포함)
2. 시스템이 Excel 데이터를 파싱하여 열(Column) 정의 추출
3. 심사위원이 각 열에 대해 점수/코멘트/체크박스 입력
4. 열 유형:
   - `score`: 점수 입력 (최대 점수 제한)
   - `comment`: 텍스트 코멘트
   - `checkbox`: 체크 여부

### 9.4 심사위원별 점수 관리

- 각 심사위원의 채점 진행 상태 추적
- 임시 저장(draft) 및 최종 제출(final) 구분
- 최종 제출 후 수정 불가
- `expertScoringSubmissions` 테이블로 제출 상태 관리

### 9.5 결과 집계

- 심사위원별 점수를 항목별로 합산
- 가중 평균 또는 단순 평균 선택
- 결과 내보내기 (Excel)
- 이벤트별 종합 결과 페이지

## 10. Rankings & Achievements

### 10.1 통합 랭킹 시스템

DAKER는 8개 카테고리의 통합 랭킹 시스템을 운영한다.

| 카테고리 | 설명 | 점수 기준 |
|---------|------|----------|
| **Total** | 종합 점수 | 모든 카테고리 합산 |
| **GitHub** | GitHub 활동 | 커밋, PR, 프로젝트 등 |
| **Raid** | 팀 활동 | Raid 참여, 기여도 |
| **Hackathon** | 해커톤 성과 | 참가, 수상 이력 |
| **Discord** | Discord 활동 | 메시지, 참여도 |
| **Community** | 커뮤니티 활동 | 포스트, 댓글, 투표 |
| **Profile** | 프로필 완성도 | 정보 입력, 인증 |
| **Growth** | 성장 지표 | 연속 접속, 활동 빈도 |

### 10.2 티어 시스템

점수에 따라 티어가 부여된다.

```
Bronze → Silver → Gold → Platinum → Diamond
```

- 각 카테고리별 독립적인 티어
- 종합(Total) 티어는 모든 카테고리의 합산 점수 기준
- 티어 승급 시 업적 알림 표시

### 10.3 레벨 & XP 시스템

- XP (경험치): 다양한 활동을 통해 획득
- 레벨업 시 업적 알림
- XP 트랜잭션 이력 관리

### 10.4 GitHub 연동

- GitHub OAuth를 통한 계정 연결
- 커밋 수, PR 수, 리포지토리 수 등 통계 자동 동기화
- GitHub 활동 기반 점수 산출
- `github-scheduler`를 통한 주기적 업데이트

### 10.5 업적 시스템 (Achievements)

```
업적 유형:
  ├── tier_upgrade: 티어 승급
  ├── level_up: 레벨업
  ├── milestone: 마일스톤 달성
  ├── first_activity: 첫 활동
  └── streak_achievement: 연속 달성
```

- 업적 달성 시 실시간 알림 (토스트/모달)
- 업적 알림 확인(acknowledge) 기능
- 프로필에 대표 배지 최대 5개 표시

### 10.6 성장 지표

`userGrowthMetrics` 테이블로 사용자의 성장을 추적한다.

- 연속 접속 일수 (현재/최대)
- 총 로그인 일수
- 주간 활동량
- 활동 이력 및 중복 방지 (일일 1회)

## 11. Admin 가이드

- 접속 URL: `https://daker.ai/admin`

`/admin` 메인 페이지에서 플랫폼 현황을 한눈에 확인할 수 있다.

### 11.4 사용자 관리

- 회원 목록: `/admin/members`
- 심사위원 관리: `/admin/expert-management`
- 사용자 역할 변경

### 11.5 콘텐츠 관리

- 커뮤니티 관리: `/admin/community-management`
- 학습 콘텐츠 관리: `/admin/learning`

- 인증서 관리: `/admin/certificates`

### 11.6 배너 관리

`/admin/banners` 에서 플랫폼 배너를 관리한다.

- 배너 생성/편집/삭제
- 배너 분석(Analytics): `/admin/banners/analytics`
- 배너 노출 기간 및 조건 설정

### 11.7 설문 관리

`/admin/surveys` 에서 설문을 관리한다.

- 설문 목록 및 상태 관리
- 설문 빌더
- 응답 분석 대시보드

### 11.8 채점 관리

`/admin/scoring` 에서 채점표를 관리한다.

- 채점표 생성 (Manual / Excel)
- 심사위원 배정
- 결과 집계 및 내보내기

### 11.9 기타 관리 기능

- 공지사항 관리: `/admin/announcements`
- 피드백 관리: `/admin/feedbacks`
- 이메일 템플릿: `/admin/email-templates`
- Discord 웹훅 설정: `/admin/discord-webhooks`
- AI 티커 설정: `/admin/ticker-settings`

## 12. Expert 가이드

### 12.1 심사위원 계정

심사위원은 관리자에 의해 등록되거나, 자가 신청을 통해 등록된다.

**심사위원 정보**:
- 이름, 소속, 전문 분야
- 역할: 심사위원, 강사, 보조강사, 멘토
- 가용 여부: 가능 / 불가능
- 전문가 태그 (기술 분야별 태그)

### 12.2 심사위원 로그인

- 접속: `https://daker.ai/expert-login`
- Google OAuth 또는 이메일/비밀번호 로그인
- 로그인 후 ExpertLayout 적용

### 12.3 평가 포털

심사위원 전용 포털에서 배정된 이벤트/해커톤의 평가를 진행한다.

**접근 경로**:
- 이벤트별 채점: `/expert/scoring`
- 해커톤 평가: `/expert/hackathon-evaluation`
- Excel 채점: `/expert/excel-scoring`
- 결과 확인: `/expert/results`

### 12.4 채점표 작성

1. 배정된 이벤트/해커톤 목록 확인
2. 발표팀 선택
3. 채점 항목별 점수 입력
4. 전체 코멘트 작성
5. 임시 저장 또는 최종 제출

**주의사항**:
- 최종 제출(`final`) 후에는 수정 불가
- 임시 저장(`draft`) 상태에서는 언제든 수정 가능
- Excel 채점의 경우 열별 필수 입력 여부 확인 필요

### 12.5 해커톤 평가 흐름
