## I. 사업 개요

### 1. 추진 목적에 대한 이해

강원대학교 **AI분야 첨단산업인재양성 부트캠프 사업단**의 비공개 알고리즘 경진대회는
부트캠프 학습 과정에서 습득한 AI·데이터 역량을 **실전 알고리즘 문제**로 검증·체화하고,
**학부와 대학원생을 구분 평가**하여 각 트랙별 우수 인재를 발굴하는 데 목적이 있습니다.

| 추진 목적 (발주처) | 본 제안의 실행 방안 |
|---|---|
| 부트캠프 교육 내용과 연계된 **실전 알고리즘 문제 경험** 제공 | 사업단 커리큘럼과 직결되는 문제 도메인 설계 (선별 과정에 사업단 승인) |
| **학부·대학원 구분** 평가로 수준별 성취도 측정 | 학부·대학원 **2개 리더보드 병행 운영** (동일 문제·별도 랭킹) |
| 부정행위 없는 **공정한 경쟁** 환경 보장 | 치팅 실시간 모니터링 + 상위 5명 수작업 코드 검증 |
| 부트캠프 **교육 효과 극대화** | 학생용 **Base-Line Code 교재** 배포로 하위권도 학습 기회 확보 |

## II. 대회 운영 절차 (핵심)

### 1. 13개 세부 업무 — 팀/식 · 주(Week) 배분

본 용역의 핵심 절차를 제안요청서 기재 순서대로 **13개 항목**으로 정리하고,
각 항목을 **설계(3주) / 운영(2주) / 검증(2주) 총 7주** 일정에 배치합니다.

| 번호 | 상세 내용 | 팀/식 | 주(Week) |
|---|---|---|---|
| **1** | **[설계]** 문제 설계 (평가 산식, 난이도, 목표, 형식, 참가자 범위, 절차 등) | 1 식 | **3 주** |
| **2** | **[설계]** Base-Line Code 설계 및 제작 | 1 식 | (설계 3주 내 병행) |
| **3** | **[설계]** 학생을 위한 교육용 Base-Line Code 교재 배포 | 1 식 | (설계 3주 내 병행) |
| **4** | **[설계]** 치팅 모니터링 (체계 구축 · 로그 수집 설계) | 1 식 | (설계 3주 내 병행) |
| **5** | **[설계]** 경진대회 페이지 제작 | 1 식 | (설계 3주 내 병행) |
| **6** | **[설계]** 명세 (규칙, 참가동의) 개발 및 제출 테스트 | 1 식 | (설계 3주 내 병행) |
| **7** | **[운영]** 대회 문의 등 CS 관리 (**주중 24시간 / 주말 48시간 이내 응답**) | 1 식 | **2 주** |
| **8** | **[운영]** 리더보드 테스트 (**학부·대학원 생 구분**) | 1 식 | (운영 2주 내 병행) |
| **9** | **[운영]** 참가자 관리 (개인 및 팀 병합, 커뮤니케이션, 부정 차단) | 1 식 | (운영 2주 내 병행) |
| **10** | **[운영]** 리더보드 (평가 산식, 스코어, 모니터링, 치팅) 관리 및 운영 | 1 식 | (운영 2주 내 병행) |
| **11** | **[검증]** 코드 치팅 및 오류 검증 (온라인 게시판 대응 포함) | 1 식 | **2 주** |
| **12** | **[검증]** 수상 및 결과 커뮤니케이션 CS 및 공지 | 1 식 | (검증 2주 내 병행) |
| **13** | **[검증]** 수상자 대상 코드 검증 (**상위 5명**) | 1 식 | (검증 2주 내 병행) |

> ※ 13개 항목은 **단계별 1식 단위 완결 과업**으로, 각 단계의 주(Week) 기간 내 병행·순차 수행됩니다. 세부 일정·착수 시점은 사업단과의 Kick-off 미팅에서 최종 확정합니다.

### 2. 7주 Gantt 구조 (단계별 흐름)

```
[ 설계 3주 ] ──────────── [ 운영 2주 ] ──────── [ 검증 2주 ]
 W1 · W2 · W3             W4 · W5              W6 · W7
 ①~⑥ 6개 세부 업무         ⑦~⑩ 4개 세부 업무     ⑪~⑬ 3개 세부 업무
```

| 단계 | 기간 | 주요 산출물 |
|---|---|---|
| **설계** | W1 ~ W3 (3주) | 문제 정의서 · 평가 산식 · Base-Line Code · 교재 · 대회 페이지 · 규칙·동의서 명세 |
| **운영** | W4 ~ W5 (2주) | CS 로그 · 학부/대학원 리더보드 · 참가자 관리 로그 · 부정 차단 기록 |
| **검증** | W6 ~ W7 (2주) | 치팅 검증 보고서 · 수상 공지 · 상위 5명 코드 재현 보고서 |

| 세부 업무 번호 | 주 투입 인력 | 지원 인력 |
|---|---|---|
| ①, ② (문제·Base-Line 설계) | **AI 엔지니어 1명** | 데이터 사이언티스트 1명 |
| ③ (교재 제작) | **기술 에디터 1명** | AI 엔지니어 검토 |
| ④, ⑩ (치팅 모니터링·리더보드) | **플랫폼 엔지니어 1명** | 총괄 PM 감독 |
| ⑤, ⑥ (대회 페이지·명세) | **프론트엔드 엔지니어 1명** | 법무·보안 검토 |
| ⑦, ⑨, ⑫ (CS·참가자 관리) | **운영 매니저 1명** | PM 일일 점검 |
| ⑧ (리더보드 테스트) | **플랫폼 엔지니어 1명** | QA 1명 |

## III. 설계 단계 상세 (W1 ~ W3 · 3주)

### 1. 문제 설계 (업무 ①)

**목표**: 부트캠프 학습 내용과 직결되며, **100명 미만 비공개 대회에서 7주 만에 완주 가능한 난이도**로 설계

| 설계 항목 | 본 제안의 접근 방법 |
|---|---|
| **평가 산식** | 문제 유형별 표준 지표(RMSE/F1/AUC/mAP 등) + 리더보드 안정성을 고려한 소수점 자리 결정 |
| **난이도** | 부트캠프 커리큘럼 진도 기반 산정 — 학부 중위권도 제출 가능, 대학원 상위권은 최적화 경쟁 |
| **목표** | 부트캠프 핵심 학습 목표(예: 정형 데이터 모델링 / CV / NLP 중 사업단 선택) 검증 |
| **형식** | Submission CSV · API 제출 · 제출 횟수 제한(일 N회) · 최종 제출 자동 선택 규칙 |
| **참가자 범위** | 부트캠프 소속 **학부생 + 대학원생** · 외부 인원 차단 (초청 링크) |
| **절차** | 등록 → 규칙 동의 → 데이터 다운로드 → 제출 → 리더보드 확인 → 본평가(Private LB) |

**Public / Private 리더보드 분할 (비공개 대회에서도 적용)**
- 대회 기간 중: Public LB (테스트셋 일부 ~30%) 실시간 공개
- 대회 종료 후: **Private LB** (미공개 테스트셋 70%) 공개 → 본 순위 확정 (과적합 방지)

### 2. Base-Line Code 설계 및 제작 (업무 ②)

- **제공 형식**: Jupyter Notebook(.ipynb) + Python 스크립트(.py) 병행
- **포함 구성**: 데이터 로드 → EDA → 전처리 → 기본 모델 학습 → 제출 CSV 생성까지 **End-to-End 실행 가능**
- **환경 명세**: Python 버전, 주요 라이브러리 버전 고정(requirements.txt), Colab 실행 가능 링크
- **성능 기준**: Base-Line 점수를 리더보드 하한선으로 공개 → 참가자의 출발점을 명확히 함

### 3. 교육용 Base-Line Code 교재 배포 (업무 ③)

**형식**: PDF 교재 + 온라인 MD 문서 병행 배포

| 교재 구성 | 설명 |
|---|---|
| 1장. 대회 이해 | 문제 정의·평가 산식 해석법 |
| 2장. 데이터 이해 | 컬럼 설명·EDA 예시·데이터 구조 다이어그램 |
| 3장. Base-Line 해설 | Base-Line Code 라인별 해설 (초급자 이해 가능 수준) |
| 4장. 성능 개선 힌트 | 하이퍼파라미터·Feature Engineering·앙상블 등 접근 방향 제시 (정답 제공 X) |
| 5장. 제출 가이드 | 제출 방법·오류 대처·FAQ |

> ※ 교재는 **학습 도우미 역할**이며 최적해를 제공하지 않습니다. 하위권 참가자도 대회를 완주할 수 있도록 설계합니다.

### 4. 치팅 모니터링 체계 구축 (업무 ④)

**설계 단계에서 운영·검증에 필요한 모니터링 인프라를 선 구축**

| 모니터링 항목 | 구현 방식 |
|---|---|
| 제출 패턴 이상 탐지 | 동일 IP / 동일 디바이스 / 단기간 다계정 제출 탐지 |
| 점수 급상승 플래그 | 비정상적 점수 상승 자동 알림 (운영자 검토 큐) |
| 계정 중복 탐지 | 이메일·디바이스 지문·세션 ID 교차 검증 |
| 외부 데이터 유입 의심 | Public LB 최상위 점수가 이론적 상한 근접 시 플래그 |

### 5. 경진대회 페이지 제작 (업무 ⑤)
