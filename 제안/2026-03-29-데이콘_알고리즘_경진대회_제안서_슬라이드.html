IV. PHASE DETAIL 4-5
PHASE DETAILS

Phase 상세 — 검증 · 시상

W7: 코드 검증
Step 1. 코드 제출 요청
상위 팀에게 학습 코드, 추론 코드, 환경 설정 파일 제출 요청 (48시간 이내)
Step 2. 재현성 검증
격리 환경에서 코드 실행, 제출 점수와 재현 점수의 오차 범위 확인 (허용 오차: ±1%)
Step 3. 부정행위 탐지
코드 유사도 분석, 외부 데이터 사용 여부, 규칙 위반 검사
Step 4. 최종 순위 확정
Private Score 기준 최종 순위 확정 및 결과 통보
W8: 결과 발표 · 시상
결과 발표
최종 순위 공개 및 수상자 안내
교내 게시판 · 학과 공지 통한 결과 공유
수상자 확정
알고리즘 점수 기반 순위 확정
대상, 최우수, 우수, 장려상 등 수상 등급 결정
시상 (교내 또는 온라인)
교내 시상식 또는 온라인 시상 운영
수상작 기술 발표 세션 포함 (선택)
결과 보고서
참가 통계, 모델 분석, SW중심대학 성과 리포트 제공

V. PLATFORM FEATURES
DAKER FEATURES

DAKER 플랫폼 핵심 기능

경진대회 운영에 필요한 모든 것을 하나의 플랫폼에서

A
대회 관리 시스템
대회 생성, 규칙 설정, 참가자 관리, 데이터셋 배포를 원클릭으로 처리하는 통합 관리 대시보드
B
실시간 리더보드
Public/Private 스코어 이원화, 실시간 순위 변동 그래프, 팀별 제출 이력 추적
C
자동 채점 서버
GPU 클러스터 기반 자동 채점, 다양한 평가 메트릭 지원 (AUC, F1, RMSE 등)
D
부정행위 탐지
코드 유사도 분석, 제출 패턴 이상 탐지, 외부 데이터 사용 여부 자동 검사
E
팀 구성 · 매칭
참가자 역량 기반 팀 빌딩, 팀 내 소통 도구, 협업 코드 관리 기능
F
채용 연계 리포트
참가자 역량 프로필, 대회 성적 기반 인재 추천, 채용 담당자 전용 리포트
