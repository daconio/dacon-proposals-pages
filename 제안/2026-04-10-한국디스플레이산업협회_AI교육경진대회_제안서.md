## 왜 데이콘인가 — 데이터 · 대회 · 교육의 차별점

개인이 데이터를 내려받아 자체 가공하는 것과 데이콘이 운영하는 것은 근본적으로 다릅니다.

### 자체 운영 vs. 데이콘 통합 운영

| 구분 | 개인 다운로드 → 자체 가공 | 데이콘 통합 운영 |
|------|------------------------|----------------|
| **라이선스** | 개인 다운로드 라이선스만 — 재배포·대회 활용 시 저작권 분쟁 리스크 | 데이콘이 원 제공기관과 대회용 라이선스 직접 협의·계약 |
| **데이터 품질** | 원본 그대로 — 결측값, 노이즈, 난이도 미검증 | 대회용 가공(비식별화, train/test 분리, 난이도 밸런싱) + 베이스라인 검증 |
| **데이터 생성** | 불가 | 데이콘 자체 제작 — 도메인 맞춤 합성 데이터 설계·구축 |
| **채점 시스템** | 자체 구축 필요 (개발 비용·기간 소요) | 자동 채점 서버 + 실시간 리더보드 즉시 제공 |
| **운영 노하우** | 대회 설계·운영 경험 부재 | 500+ 대회 운영 — 평가지표 설계, 부정행위 방지, 일정 관리 |
| **교육 연계** | 별도 교육 플랫폼 필요 | DAKER 교육 → DACON 대회 원스톱 연계 |

### 데이콘 데이터 구축 역량 — 실제 사례

| 유형 | 대회명 | 내용 |
|------|--------|------|
| **데이콘 자체 제작** | 월간 데이콘 반도체 박막 두께 분석 경진대회 | 데이콘이 직접 설계·생성한 데이터 (dacon.io/competitions/official/235554) |
| **원본 → 대회용 변환** | 스마트 해운물류 x AI 미션 챌린지 | 이상신호 감지 기반 비정상 작동 진단 — 원본 데이터를 대회 목적에 맞게 재구축 (dacon.io/competitions/official/236590) |
| **협회 데이터 가공** | (디스플레이 도메인 맞춤 구축 가능) | 협회·회원사 데이터 비식별화·가공 또는 도메인 특성 반영 합성 데이터 생성 |

## 1. 제안 개요

### 배경

한국디스플레이산업협회의 '한국디스플레이아카데미' 교육 과정의 일환으로, **AI 교육 → 경진대회**로 이어지는 통합 프로그램을 제안합니다. 교육에서 배운 AI 역량을 경진대회에서 실전 적용하는 구조로, 참가자가 디스플레이·반도체 도메인의 실제 데이터를 다루며 문제 해결 능력을 키우는 것이 핵심입니다.

### 핵심 정보

| 항목 | 내용 |
|------|------|
| **프로그램** | AI 교육 + 경진대회 통합 운영 |
| **주최** | 한국디스플레이산업협회 |
| **운영** | 데이콘(DACON) |
| **대상** | 한국디스플레이아카데미 교육생 |
| **구성** | Phase 1. AI 실습 교육 → Phase 2. AI 경진대회 |
| **데이터** | 오픈 데이터 기반 — 디스플레이·반도체·제조 도메인 |
| **플랫폼** | 교육: DAKER(daker.ai) / 대회: DACON(dacon.io) |

### 프로그램 전체 흐름

```

(DAKER 플랫폼)                     (DACON 플랫폼)

     ↓                                   ↓

     ↓                                   ↓

     ↓                                   ↓
[경진대회 준비 워크숍]               [최종 평가 & 시상]
```

## 2. 프로그램 설계 철학

### 왜 "교육 + 대회" 통합인가

AI 교육만으로는 실전 역량을 기르기 어렵고, 경진대회만으로는 기초가 부족한 참가자가 진입하기 어렵습니다. 두 가지를 하나의 흐름으로 연결하면, 교육에서 습득한 지식을 대회에서 바로 검증하고 심화할 수 있습니다.

| 구분 | 교육만 진행 | 대회만 진행 | 교육 + 대회 통합 |
|------|-----------|-----------|----------------|
| 실전 경험 | 부족 | 풍부하나 진입 장벽 높음 | 교육이 대회 준비 역할 |
| 동기 부여 | 수료 후 동력 저하 | 높으나 초보자 이탈 | 대회 목표가 학습 동기 |
| 성과 측정 | 만족도 설문 | 리더보드 순위 | 학습 진도 + 대회 성과 |
| 도메인 연결 | 범용 예제 | 도메인 데이터 | 교육부터 도메인 데이터 활용 |

## 3. Phase 1 — AI 실습 교육

> 경진대회 참여에 필요한 AI 기초 역량을 체계적으로 갖추는 과정

### 교육 커리큘럼 (안)

| 모듈 | 내용 | 시간 |
|------|------|------|
| **모듈 1. AI·머신러닝 기초** | 머신러닝 개념, 지도학습/비지도학습, 평가 지표(정확도, F1, AUC 등) | 3시간 |
| **모듈 2. 데이터 전처리 실습** | Pandas 기반 데이터 탐색(EDA), 결측값 처리, 특성 엔지니어링 | 3시간 |
| **모듈 3. 도메인 특화 — 이상 탐지** | 제조·디스플레이 공정 데이터의 특성, 이상 탐지(Anomaly Detection) 기법 | 3시간 |
| **모듈 4. 모델링 실습** | scikit-learn, XGBoost, LightGBM 등 알고리즘별 모델 구축·튜닝 | 3시간 |
| **모듈 5. 딥러닝 기초** | CNN/오토인코더 기반 이미지·시계열 이상 탐지 (선택) | 3시간 |
| **모듈 6. 경진대회 준비 워크숍** | DACON 플랫폼 사용법, 제출 형식, 리더보드 전략, 앙상블 기법 | 2시간 |

> 총 교육 시간 및 일정은 협의하여 조정 가능합니다.

### 교육 운영 방식

| 항목 | 내용 |
|------|------|
| **형태** | 이론 30% + 실습 70% |
| **실습 환경** | DACON Jupyter LAB (클라우드 기반, 별도 설치 불요) |
| **실습 데이터** | 경진대회에서 사용할 데이터의 샘플 버전으로 실습 — 교육과 대회의 연속성 확보 |
| **강사** | 데이콘 소속 AI 전문 강사 + 보조 강사 |
| **학습 플랫폼** | DAKER — 교재 배포, 팀 구성, 학습 진도 트래킹 |

## 4. Phase 2 — AI 경진대회

> 교육에서 습득한 역량을 실전 데이터로 검증하는 경진대회

### 대회 설계 (안)

| 항목 | 내용 |
|------|------|
| **대회명 (안)** | 한국디스플레이아카데미 AI Challenge |
| **유형** | AI 알고리즘 대회 (자동 채점) |
| **플랫폼** | DACON (dacon.io) |
| **참가 대상** | 한국디스플레이아카데미 교육생 (비공개 대회 가능) |
| **기간** | 2~4주 (협의) |
| **평가** | 자동 채점 + 실시간 리더보드 |

### 대회 주제 후보

디스플레이·반도체·제조 도메인의 오픈 데이터를 활용한 실습형 대회를 기획합니다.

| 주제 | 데이터 유형 | 난이도 | 관련 레퍼런스 |
|------|-----------|--------|-------------|
| **디스플레이 패널 결함 탐지** | 이미지(패널 표면) 또는 센서 로그 | 중급 | 디스플레이 도메인 특화 |
| **반도체 소자 이상 탐지** | 센서 데이터(시계열) | 중급 | DACON 기 개최 대회 |
| **제조 공정 품질 예측** | 공정 파라미터(정형 데이터) | 초~중급 | 제조 도메인 범용 |
| **설비 예지 보전(PHM)** | 진동·온도 센서 시계열 | 중~고급 | 스마트 팩토리 |

> 최종 주제는 협회의 교육 목표와 참가자 수준을 고려하여 함께 결정합니다.

### 대회 운영 흐름

| 단계 | 기간 | 내용 |
|------|------|------|
| **대회 공고** | D-7 | 대회 페이지 개설, 규칙·평가 기준 안내 |
| **데이터 공개** | D-day | 학습 데이터(train) + 테스트 데이터(test) 배포 |
| **대회 진행** | 2~4주 | 참가자 모델 개발 → 예측 결과 제출 → 자동 채점 → 리더보드 갱신 |
| **중간 피드백** | 대회 중 | 상위 솔루션 힌트 공유, Q&A 세션 (선택) |
| **최종 평가** | 마감일 | Private 리더보드 공개 (최종 순위 확정) |
| **시상 & 공유** | D+3 | 수상자 발표, 우수 솔루션 공유 세션 |

## 5. 데이터셋 기획 및 제공

### 데이터 확보 방안

데이콘은 500+ 대회 운영 경험을 바탕으로, 디스플레이·반도체 도메인에 적합한 데이터를 확보·가공할 수 있습니다.

| 방안 | 설명 | 특징 |
|------|------|------|
| **기존 대회 데이터 재활용** | 반도체 소자 이상 탐지, 스마트 해운물류 등 기 개최 대회 데이터 | 검증된 품질, 빠른 적용 |
| **오픈 데이터 큐레이션** | 공개 데이터셋(UCI, Kaggle, 공공데이터)에서 디스플레이·제조 관련 데이터 선별·가공 | 저작권 이슈 없음 |
| **협회 보유 데이터 가공** | 협회·회원사가 제공하는 데이터를 대회용으로 비식별화·가공 | 도메인 적합도 최고 |
| **합성 데이터 생성** | 실제 공정 패턴을 반영한 합성(Synthetic) 데이터 생성 | 보안 이슈 없음 |

### 데이터 제공 절차

```
1. 대회 주제 확정
   ↓
2. 데이터 후보 선정 (기존 대회 / 오픈 데이터 / 협회 제공)
   ↓
3. 데이터 가공 — 비식별화, 품질 검증, train/test 분리, 평가 지표 설계
   ↓
4. 사전 테스트 — 베이스라인 모델로 대회 난이도 검증
   ↓
5. 데이터 확정 & 대회 적용
```

## 6. DACON 플랫폼 — 경진대회 운영

### 자동 채점 경진대회 시스템

DACON은 대한민국 최대 AI 경진대회 플랫폼으로, 대회 운영에 필요한 모든 인프라를 제공합니다.

| 기능 | 설명 |
|------|------|
| **자동 채점 서버** | 참가자가 예측 결과(CSV)를 제출하면 즉시 자동 채점 — 정확도, F1, RMSE 등 다양한 평가 지표 지원 |
| **실시간 리더보드** | Public 리더보드(대회 중 실시간 순위) + Private 리더보드(최종 평가용, 대회 종료 시 공개) |
| **Jupyter LAB** | 클라우드 기반 코딩 환경 — 별도 설치 없이 브라우저에서 바로 모델 개발 |
| **토론 게시판** | 참가자 간 질의응답, 인사이트 공유, 운영진 공지 |
| **코드 공유** | 참가자의 코드·노트북 공유 — 베이스라인 코드 제공 가능 |
| **비공개 대회** | 초대 코드 기반 비공개 운영 가능 — 교육생만 참여하는 폐쇄형 대회 |

### 대회 유형 선택

| 유형 | 제출 방식 | 적합 상황 |
|------|----------|----------|
| **결과 제출형** | 예측 결과 CSV 제출 → 자동 채점 | 초~중급, 교육 연계 대회에 적합 |
| **코드 제출형** | 코드 자체를 제출 → 서버에서 실행·채점 | 중~고급, 재현성 중시 |

> 교육 연계 대회에는 **결과 제출형**을 추천합니다. 참가자가 채점 결과를 즉시 확인하며 모델을 개선할 수 있어 학습 효과가 높습니다.
