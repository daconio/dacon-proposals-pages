## 2. 프로그램 설계 철학

### 왜 "결과물 중심"인가

기존 AI 교육의 한계는 명확합니다. 이론 강의 80% + 따라하기 실습 20%의 구조에서는 교육이 끝나는 순간 내용이 휘발됩니다. 본 프로그램은 이 비율을 뒤집습니다.

| 기존 AI 교육 | 본 프로그램 |
|-------------|-----------|
| "AI가 무엇인지" 설명 | "AI로 무엇을 만들 수 있는지" 체험 |
| 범용 예제로 따라하기 | 각자의 리서치 주제로 직접 구축 |
| 교육 종료 = 끝 | 교육 종료 = 업무 적용 시작 |

### 금융 리서치에 AI가 필요한 이유

금융 리서치 업무는 **수집 → 정제 → 분석 → 보고서 작성**이라는 반복적 파이프라인으로 구성됩니다. 이 과정에서 AI 에이전트는 마치 숙련된 리서치 어시스턴트처럼 작동합니다.

- **정보 수집**: 뉴스, 공시, 시장 데이터를 구조화된 형태로 자동 정리
- **신뢰성 필터링**: 출처·시점·일관성 기준으로 정보의 품질을 자동 판별
- **요약·인사이트 추출**: 대량의 텍스트에서 핵심 시사점만 압축
- **보고서 자동 생성**: 조직 고유의 양식에 맞춘 리포트 초안 작성

팀장급 리더가 이 도구를 직접 설계·운용할 수 있으면, 팀 전체의 리서치 생산성이 비약적으로 향상됩니다.

## 6. AI 도구: Claude 상세 안내

### 6-1. Claude란?

Claude는 Anthropic이 개발한 AI 어시스턴트로, 대화형 인터페이스를 통해 텍스트 분석, 요약, 작성, 코드 실행 등을 수행합니다. 비유하자면, **24시간 대기하는 리서치 어시스턴트**가 한 명 더 생기는 것과 같습니다. 다만, 이 어시스턴트는 지시를 얼마나 정확하게 하느냐(프롬프트 설계)에 따라 결과물의 품질이 극적으로 달라집니다 — 이것이 본 교육의 핵심입니다.

### 6-5. Claude 사용 방법 — 교육 당일 기준

교육 당일 참가자가 Claude를 사용하는 흐름은 다음과 같습니다.

**Step 1. 접속**
- 데이콘이 사전 발송한 초대 이메일의 링크를 클릭하여 Claude Code Teams 조직에 가입
- 웹 브라우저에서 [claude.ai](https://claude.ai) 접속 또는 Claude Code 데스크탑 앱 실행

**Step 2. 프로젝트 진입**
- 데이콘(관리자)이 사전 생성한 "금융 리서치 워크숍" 프로젝트에 입장
- 프로젝트 내 **지식 베이스**에 업로드된 샘플 데이터·프롬프트 템플릿 확인

**Step 3. 대화 시작**
- 채팅창에 자연어로 지시 입력 (예: "최근 1주일 반도체 섹터 주요 뉴스를 수집하고, 투자 시사점 3가지로 요약해줘")
- Claude가 응답 생성 → 결과 확인 → 프롬프트 수정 → 재실행 (피드백 루프)

**Step 4. Artifacts 활용**
- Claude의 **Artifacts** 기능으로 표, 차트, 코드를 즉석에서 생성·수정
- 생성된 보고서를 복사하여 사내 보고 양식에 붙여넣기

**Step 5. 결과물 저장·공유**
- 완성된 프롬프트 세트와 워크플로우를 프로젝트 내에 저장
- 팀원 간 결과물 상호 열람 및 피드백
