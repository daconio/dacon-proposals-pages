# DAKER × DACON — 플랫폼 단계별 통합 계획서

> daker.ai와 dacon.io를 하나의 에코시스템으로 합쳐, AI 대회 참가자와 AI 협업 팀 빌더가 동일한 플랫폼 위에서 활동하도록 한다.

**2026.04**

---

## I. 개요

daker.ai(클로드 코드 기반 차세대 AI 협업·팀 빌딩 플랫폼)와 dacon.io(기존 AI·데이터 사이언스 경진대회 플랫폼)를 단계적으로 통합한다.

두 플랫폼은 상호 보완적 사용자 군을 보유한다. dacon.io의 대회 참가자가 daker.ai에서 팀을 결성하고, 팀을 갖춘 상태로 dacon.io 대회에 제출하는 흐름을 자연스럽게 연결하는 것이 핵심 목표다.

---

## II. 현재 상태 분석

| 항목 | daker.ai | dacon.io |
|------|---------|---------|
| 프레임워크 | React + Vite | Vue.js + Vuetify |
| 스타일링 | CSS-in-JS / 자체 디자인 시스템 | Tailwind CSS |
| 실시간 협업 | y-websocket | — |
| UI 특성 | 다크모드, 팀 빌딩 중심 | 경진대회/리더보드/티어 시스템 |
| 분석 도구 | GA4 | GA4 / GTM / Hotjar |
| 인증 | 자체 계정 | 자체 계정 |

---

## III. 통합 원칙

1. **Non-disruptive** — dacon.io 기존 UX를 유지하며 점진적으로 통합한다
2. **Feature-flagged** — 모든 통합 기능은 on/off 가능하게 설계한다
3. **API-first** — UI 통합 전에 API 계층을 선행 구축한다
4. **Single source of truth** — 계정·대회 데이터는 dacon.io가 마스터다
5. **Reversible** — 각 단계는 독립적으로 롤백 가능하다

---

## IV. 6단계 통합 스프린트

### Sprint 1 — 브랜드 진입점 Cross-linking `(1~2주)`

두 플랫폼 간 가장 가볍고 안전한 첫 연결 고리를 만든다.

| 작업 항목 | 상세 |
|----------|------|
| GNB 메뉴 추가 | dacon.io 글로벌 내비게이션에 "팀 빌딩(daker.ai)" 항목 추가 |
| CTA 추가 | daker.ai 랜딩페이지에 "대회 참가하기" 버튼 → dacon.io 연결 |
| 공통 브랜드 토큰 | 색상·폰트 가이드라인 정렬 |
| OG 메타태그 통일 | 양 플랫폼 소셜 공유 미리보기 포맷 통일 |
| Feature Flag 적용 | 조건부 렌더링으로 안전하게 배포 |

---

### Sprint 2 — OAuth SSO 계정 통합 `(3~4주)`

단일 계정으로 양 플랫폼에 접근할 수 있도록 인증 레이어를 구축한다.

| 작업 항목 | 상세 |
|----------|------|
| OAuth 2.0 + PKCE 확장 | dacon.io를 인증 공급자(Authorization Server)로 확장 |
| 엔드포인트 구현 | `/oauth/authorize`, `/oauth/token`, `/oauth/userinfo` |
| daker.ai 로그인 연동 | "dacon 계정으로 로그인" 버튼 추가 |
| 계정 마이그레이션 | 기존 daker.ai 단독 계정 → dacon 계정 연결 플로우 |
| JWT 토큰 정책 | 만료 1시간, Refresh Token 갱신 |

---

### Sprint 3 — 대회 데이터 API 연동 `(2~3주)`

dacon.io의 대회 데이터를 daker.ai에서 조회하고 활용할 수 있도록 API를 개방한다.

| API | 설명 |
|-----|------|
| `GET /api/v1/competitions?status=active` | 진행 중인 대회 목록 조회 |
| `GET /api/v1/competitions/{id}` | 대회 상세 정보 조회 |

**연동 기능**
- daker.ai 팀 빌딩 시 "참가 대회 선택" 드롭다운 연결
- dacon.io 대회 상세 페이지에 "연결된 팀" 카운트 표시
- Circuit Breaker + Rate Limiting 적용 (장애 전파 방지)

---

### Sprint 4 — UX Cross-navigation `(2~3주)`

사용자가 두 플랫폼 사이를 자연스럽게 이동하도록 UX 동선을 연결한다.

| 작업 항목 | 상세 |
|----------|------|
| 팀 구하기 진입 | dacon.io 대회 상세 → "팀 구하기(daker.ai)" 버튼 |
| 대회 제출 딥링크 | daker.ai 팀 완료 후 "대회 제출하기(dacon.io)" 딥링크 |
| 공통 헤더 컴포넌트 | Web Component 기반 공유 헤더 (React·Vue 모두 탑재 가능) |
| 마이페이지 연동 | dacon.io 마이페이지에 daker.ai 팀 참여 이력 표시 |
| 모바일 대응 | 반응형 CTA 처리 |

---

### Sprint 5 — 통합 프로필·알림·Analytics `(3~4주)`

두 플랫폼의 사용자 데이터를 합산해 통합 정체성과 알림 경험을 제공한다.

| 작업 항목 | 상세 |
|----------|------|
| 통합 프로필 API | dacon 티어 + daker 활동 점수를 결합한 단일 프로필 응답 |
| 수상 → 배지 자동 부여 | 대회 수상 시 daker.ai 팀 배지 자동 부여 (Webhook) |
| 통합 알림 | 대회 마감 / 팀 초대 / 리더보드 변동 알림 통합 |
| 크로스 플랫폼 분석 | GTM 기반 이벤트 스키마 통일 |
| 통합 검색 | 대회·팀·사용자 통합 검색 진입점 |

---

### Sprint 6 — 어드민 통합 및 운영 프로세스 `(2~3주)`

플랫폼 운영자가 단일 콘솔에서 양 플랫폼을 관리할 수 있도록 어드민을 통합한다.

| 작업 항목 | 상세 |
|----------|------|
| 통합 어드민 콘솔 | 사용자/대회/팀 통합 조회 및 관리 |
| 주최사 대시보드 | 대회 주최사 화면에 팀 빌딩 현황 위젯 추가 |
| 공유 블랙리스트 | 어뷰징 사용자 공동 차단 메커니즘 |
| 통합 헬프센터 | dacon.io/help 단일 진입점으로 통합 |
| 장애 대응 RunBook | SLA 정의 및 플랫폼 간 장애 전파 대응 절차 문서화 |

## V. 기술 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                      사용자 브라우저                      │
│                                                         │
│   daker.ai (React + Vite)   dacon.io (Vue.js + Vuetify) │
│   ┌─────────────────────┐   ┌──────────────────────┐   │
│   │  Web Component 헤더  │   │  Web Component 헤더  │   │
│   └──────────┬──────────┘   └──────────┬───────────┘   │
└──────────────┼──────────────────────────┼───────────────┘
               │                          │
               ▼                          ▼
        ┌──────────────────────────────────┐
        │           API Gateway            │
        │  (Circuit Breaker / Rate Limit)  │
        └───────────┬──────────────────────┘
                    │
         ┌──────────┴──────────┐
         ▼                     ▼
  daker.ai API Server   dacon.io API Server
  (팀/협업 마스터)        (계정/대회 마스터)
                              │
                    OAuth 2.0 + PKCE
                    JWT (만료 1시간)
```

**핵심 기술 결정**

| 항목 | 결정 사항 | 이유 |
|------|----------|------|
| 컴포넌트 공유 | Web Components | React·Vue 간 프레임워크 격리 |
| 인증 표준 | OAuth 2.0 + PKCE | 보안 표준, 확장 용이 |
| 장애 격리 | Circuit Breaker | 단일 장애가 전체로 전파되지 않도록 |
| 독립 배포 | API Gateway 계층 | 양 플랫폼 독립 릴리즈 유지 |
| 사용자 추적 | GTM 크로스 플랫폼 스키마 | 기존 Hotjar/GA4 인프라 재사용 |
