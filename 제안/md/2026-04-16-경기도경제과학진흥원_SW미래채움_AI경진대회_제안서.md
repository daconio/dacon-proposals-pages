## III. 사업 이해 — SW미래채움 경진대회 설계

### III-1. SW미래채움 사업 목적 연계

SW미래채움 사업은 소프트웨어 분야의 미래 인재를 발굴·육성하는 것을 목표로 합니다. 본 경진대회는 이 취지를 세 가지 축으로 구현합니다.

| 사업 목적 | 대회 연계 설계 |
|-----------|----------------|
| SW 인재 저변 확대 | 고등학생 대상 3트랙으로 진입 장벽 최소화 |
| 실전 역량 배양 | 아이디어 → 데이터 분석 → 서비스 개발까지 단계별 경험 |
| 산학 연계 강화 | 7개 대학 대학생 멘토가 직접 지도, 대학 예선 연계 |

### III-2. 3트랙 Overview

| 트랙 | 유형 | 핵심 역량 | 난이도 | 산출물 |
|------|------|-----------|--------|--------|
| **Track A — 아이디어톤** | 기획 중심 | 문제 정의, 서비스 기획, PT 발표 | ★☆☆ | 기획서(PPT) + 발표 영상 |
| **Track B — 데이터톤** | 분석 중심 | 데이터 전처리, 시각화, 인사이트 도출 | ★★☆ | 분석 노트북 + 리더보드 점수 |
| **Track C — AI서비스톤** | 개발 중심 (바이브코딩) | 프롬프트 설계, AI 코드 생성, MVP 개발 | ★★★ | 실제 작동 웹 서비스(MVP) |

세 트랙은 **독립 참가** 또는 **복수 참가**가 가능하며, 각 트랙별로 대학 예선 → 본선 → 결선 구조를 동일하게 적용합니다.

## VI. Track C — AI서비스톤(바이브코딩) 상세

### 바이브코딩이란?

2025년 Andrej Karpathy가 명명하고 Collins Dictionary 2025 올해의 단어로 선정된 새로운 소프트웨어 개발 패러다임입니다. 자연어로 의도를 설계하면 AI가 코드를 생성하는 방식으로, 코딩 경험이 없는 고등학생도 실제 작동하는 웹 서비스(MVP)를 만들 수 있습니다.

### 참가 대상 및 주제

| 항목 | 내용 |
|------|------|
| 참가 대상 | 경기도 내 고등학생 (2~4인 팀) |
| 대회 주제 | AI로 만드는 우리 학교 / 우리 동네 서비스 |
| 핵심 산출물 | 실제 작동하는 웹 서비스(MVP) + 기획서 |
| 코딩 경험 | **불필요** — 바이브코딩(자연어 → AI 코드 생성)으로 개발 |

### DAKER 활용 기능

| 기능 | 활용 방식 |
|------|-----------|
| **트랙 러닝** | 바이브코딩 기초 → 프롬프트 설계 → MVP 개발 실습 커리큘럼 |
| **작전실(War Room)** | 팀 전용 협업 공간, 코드·링크·산출물 공유, 멘토링 로그 자동 저장 |
| **AI 멘토링** | 프롬프트 최적화 가이드, 기술 스택 추천, 디버깅 지원 |
| **대학생 멘토** | 서비스 아키텍처 설계, 프롬프트 엔지니어링, 배포 지원 |
| **제출 포털** | MVP 배포 URL + 소스코드 + 기획서 업로드, 자동 유효성 검증 |
| **ELO 1:1 아레나** | 참가자 간 상호 블라인드 평가 (30% 반영) |
| **전문가 심사** | 심사위원 6~7인 개별 배정, 루브릭 기반 평가 (70% 반영) |
| **갤러리** | MVP 자동 게시, 스크린샷 미리보기, 영구 포트폴리오 |

## VII. 대학 연계 멘토링 시스템

### VII-1. 7개 대학 멘토링 구조

| 구분 | 내용 |
|------|------|
| 참여 대학 | 경기도 내 7개 대학 (SW·AI·데이터 관련 학과) |
| 멘토 규모 | 대학당 5~7명, 총 **35~49명** |
| 멘토 자격 | 관련 학과 3학년 이상 또는 대학원생, SW·AI 프로젝트 경험자 |
| 멘토링 방식 | 온라인(DAKER 1:1 예약) + 오프라인(대학 예선 현장) |
| 멘토링 횟수 | 팀당 **4회** (2주 간격, 예선 기간 중) |

### VII-2. DAKER 멘토 관리 기능

| 기능 | 설명 |
|------|------|
| **캘린더 슬롯 관리** | 멘토별 가용 시간 등록, 고등학생 팀 1:1 예약 |
| **하이브리드 예약** | 온라인(화상) / 오프라인(대학 방문) 선택 가능 |
| **쿼터 관리** | 멘토당 주간 최대 세션 수 설정, 과부하 방지 |
| **멘토링 로그** | 세션 요약·피드백·액션 아이템 자동 저장 |
| **운영자 대시보드** | 멘토링 진행률, 팀별 참여 현황 실시간 모니터링 |

### VII-3. 멘토링 운영 수학

| 항목 | 산출 |
|------|------|
| 예상 참가팀 | 약 150팀 (3트랙 합산) |
| 멘토 42명 | 멘토 1인당 약 3.6팀 담당 |
| 4회차 멘토링 | 150팀 × 4회 = 총 600 세션 |
| 일일 세션 | 멘토당 1~2건/일 (주 5일 기준) |
| 24시간 AI 보조 | 심야·주말에도 AI Office Hours로 멘토링 공백 제로 |

### VII-4. AI 멘토링 — Forcing Questions 프레임워크

DAKER AI 멘토는 6가지 핵심 질문으로 고등학생의 아이디어를 구체화합니다.

| 단계 | 질문 | 목적 |
|------|------|------|
| 1 | **수요 현실성** — 이 문제를 정말 겪는 사람이 있나? | 실제 수요 검증 |
| 2 | **현재 상태** — 지금은 이 문제를 어떻게 해결하나? | 기존 대안 분석 |
| 3 | **절박한 구체성** — 가장 아픈 사람은 누구인가? | 타겟 사용자 특정 |
| 4 | **가장 좁은 쐐기** — 가장 작은 MVP는 무엇인가? | 최소 기능 정의 |
| 5 | **관찰과 놀라움** — 직접 관찰하니 무엇이 달랐나? | 현장 인사이트 |
| 6 | **미래 적합성** — 3년 후에도 의미 있는 서비스인가? | 지속 가능성 점검 |
