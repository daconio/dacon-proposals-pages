## 왜 DAKER인가 — Executive Summary

본 공모전은 **DAKER 플랫폼의 자동화 기능**을 적극 활용하여 데이콘 운영 인력 투입을 최소화하면서도 14만 AI 커뮤니티 기반의 폭넓은 참가자 모집과 안정적인 대회 운영을 동시에 달성하는 것을 핵심으로 합니다.

### 자체 운영 vs DAKER 통합 운영 (인력투입 비교)

| 운영 항목 | 일반 자체 운영 | DAKER 자동화 운영 |
|----------|---------------|-----------------|
| 대회 페이지 제작 | 별도 디자이너·개발자 소요 | DAKER 페이지 빌더 — 운영자 1명 셋업 |
| 제출/저장/채점 | 수기 처리 또는 별도 시스템 구축 | DAKER 자동 제출·저장·로그 내장 |
| 리더보드 운영 | 수동 집계·갱신 | 실시간 자동 산식 적용·갱신 |
| 부정행위 차단 | 사람이 모니터링 | 중복 제출·동일 IP·치팅 자동 탐지 |
| 피어 평가(예선) | 수동 매칭·집계 | ELO 알고리즘 기반 자동 매칭·점수화 |

## 4. DAKER 자동화 — 데이콘 인력투입 최소화 전략

DAKER 플랫폼의 자동화 기능으로 통상 5~7명이 투입되는 공모전을 **2~3명** 운영 체제로 효율화합니다.

### 자동화 영역 매핑

| 자동화 영역 | DAKER 기능 | 절감 효과 |
|------------|-----------|----------|
| **참가 등록 · 팀 관리** | 셀프 회원가입 + 팀 빌딩 + 팀원 병합 자동 처리 | 사람 손 거의 불필요 |
| **제출 · 저장** | 코드/URL/영상 업로드 자동 검증 + 무중단 저장 | 별도 제출 관리자 불필요 |
| **리더보드** | 평가 산식 입력 → 실시간 자동 스코어링·갱신 | 수동 집계·갱신 작업 0 |
| **부정행위 차단** | 동일 IP·중복 제출·외부 코드 복제 자동 탐지 + 운영자 알림 | 24시간 모니터링 인력 불필요 |
| **피어 평가** | ELO 기반 1:1 매칭 자동 배포 + 점수 수렴 | 매칭·집계 작업 0 |
| **심사위원 업보트** | 심사위원 전용 대시보드에서 후보 작품 자동 분배 + 업보트 집계 | 결과 정리 자동화 |
| **CS · 알림** | FAQ 자동 응답 + 문의 큐 + 결과 일괄 메일/문자 | CS 인력 1명 → 주중 비상 대응만 |
| **결과 통보** | 수상/탈락 결과 일괄 발송 + 참가증 자동 발급 | 수동 발송 작업 0 |
