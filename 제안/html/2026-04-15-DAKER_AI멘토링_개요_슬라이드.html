PROBLEM / SOLUTION

기존 해커톤 멘토링의 4가지 한계를 동시에 해결

문제
  • 멘토 역량 편차 — 같은 CP를 받아도 팀별 품질 격차
  • 사각지대 — 스코프·리스크가 본선 직전까지 노출 안 됨
  • 멘토 부족 — 본선 집중 시간대 기술 자문 과부하
  • B2B 상품화 공백 — DAKER 플랫폼 부가가치 창출 여지
해결
  • 품질 균등화 — 5종 시스템 프롬프트로 고정 품질
  • 구조화 발견 — 강제 질문 흐름이 스코프/리스크 노출
  • 24/7 접근 — AI 멘토 상시 가용, SSE 실시간 응답
  • B2B 상품화 — 멘토링 모드 설정 · 비용 한도 관리
핵심 인사이트

“사람 멘토의 보조”가 아니라 “구조화된 AI 세션 + 사람 멘토 하이브리드”를 전제로 설계.
AI가 먼저 스코프·리스크를 노출시키고, 사람 멘토는 고차원 판단에 집중.

에러 분기
  • 403 CONSENT_REQUIRED → 서버가 재차 동의 요구 → ConsentDialog 재오픈
  • 409 Conflict → 동일 CP에 진행 중 세션 존재 → data.sessionId로 기존 세션 이어가기
  • 429 → Rate Limit · 일일 토큰 초과 → 사용자 안내
pendingAction 패턴
동의 모달이 열리기 전 사용자가 시도한 행동(CP 시작 · 자유 채팅)을 pendingAction state에 저장.
동의 완료 후 자동으로 해당 행동을 재실행 — 흐름 끊김 없이 이어가기.
