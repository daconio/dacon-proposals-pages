III. MISSION CRITICALDACON
MISSION CRITICAL
Zero Tolerance

단 한 치의 오차도 허용되지 않는 현장 라이브 해커톤

참가자 150명이 The K Hotel 한 공간에서 정해진 2시간을 동시에 경쟁하는 행사는
네트워크 지연·시스템 오류 발생 시 시간을 연장하거나 유연하게 대처하기 어려운 물리적 한계가 있습니다.

온라인 해커톤
• 장애 시 시간 연장 가능
• 분산된 동시 접속
• 재시도 안내로 수습 가능
• 목표: 평균 가용성 99%
현장 라이브 해커톤
• 연장 불가 — 참가자 전원 현장 대기 중
• 초단위 동시 스파이크 (전원 동시 시작·제출)
• 수백 명의 현장 불만 즉시 발생
• 목표: 무결성 100% · 모든 제출 0 누락
“실시간 리더보드 채점과 API 연동에 단 한 치의 오차도 발생해서는 안 되는 미션 크리티컬 업무”
— 150명 동시 트래픽을 견디며, 제출 누락·순서 오류·점수 불일치가 0건이어야 합니다

III. MISSION CRITICALDACON
SIX REQUIREMENTS

6대 미션 크리티컬 요건

150명 동시 경쟁을 무결성 100%로 완수하기 위한 6가지 핵심 요건

01
동시 트래픽 내성
150명 동시 접속·동시 제출 시점의 초단위 트래픽 스파이크를 견디는 서버 인프라 (자동 확장 + 핫스탠바이)
02
실시간 리더보드 무오차
모든 제출이 누락·중복·순서 오류 없이 실시간 리더보드에 반영. 채점 엔진 이중화, 5분 간격 정합성 검증
03
LLM API 연동 안정성
ChatGPT 등 외부 LLM API의 rate limit, 타임아웃, 429 오류를 사전 방어. Fallback Key 풀 + 재시도 큐
04
현장 네트워크 이중화
The K Hotel Wi-Fi + 백업 회선(4G/5G 라우터) 병행 운영. 네트워크 단일 장애점 제거
05
제로 데이터 손실
제출 직후 자동 스냅샷, 로컬 버퍼 + 서버 이중 저장. 네트워크 일시 단절 시에도 제출 보존
06
현장 즉시 대응 인력
운영 PM 1명 + 현장 엔지니어 2명 + 온라인 백업 엔지니어 1명 상시 실시간 모니터링

장애 시나리오 대응 + 5회 필수 사전 테스트

사전에 예상 가능한 모든 장애를 훈련하고, 실 운영 환경과 동일한 부하로 검증합니다

구분예상 장애대응 전략

S2ChatGPT API 429/500Fallback Key 풀 전환, 재시도 큐 자동 처리
S3현장 Wi-Fi 다운백업 4G/5G 회선 자동 전환 + 로컬 저장
S4리더보드 점수 불일치채점 엔진 이중화, 5분 정합성 검증, 즉시 알림
사전 테스트 5회 — 모두 필수
회차목적확인 항목
1차데이터 무결성결측값, 포맷, 가명화 누락
2차평가 산식 검증점수 범위, 엣지케이스
3차플랫폼 연동제출→채점→리더보드 흐름
4차부하 테스트150명 동시 접속·제출 내성
5차현장 최종 리허설The K Hotel 현장 동일 환경 훈련
부하 테스트 4차와 현장 리허설 5차는 다른 대회와 차별화된 필수 절차입니다 — 150명 동시 접속·LLM API 동시 호출 내성을 실 운영 전에 반드시 검증합니다
