    개요

    1.1 비전

    DAKER LMS는 AI 기반 맞춤형 학습 관리 시스템으로, 기업·대학·공공기관의 AI/데이터 교육을 위한 올인원 플랫폼이다. 단순 콘텐츠 전달을 넘어 AI 교재 자동 생성, 실시간 학습 분석, 게이미피케이션, 팀 학습 연계를 통합하여 학습 효과를 극대화한다.

    1.2 핵심 가치

    가치설명
    AI-Native 교재 생성Gemini + GPT 상호 검수 파이프라인으로 교재를 자동 생성. 관리자의 교재 제작 부담을 90% 이상 절감
    다양한 학습 방식마크다운 콘텐츠, 퀴즈, 프롬프트 실습, 플래시카드, PDF, 분기 학습 등 6종 스테이지 유형 지원
    데이터 기반 분석xAPI 이벤트 스트림 기반 실시간 학습 행동 추적 및 12개 분석 대시보드
    게이미피케이션Apple Watch 스타일 활동 링, 스트릭, 배지, 리더보드로 학습 동기 부여
    해커톤/Raid 연계DAKER의 해커톤·Raid 플랫폼과 유기적으로 연결된 학습 트랙 시스템

    1.3 대상 사용자

    사용자 유형역할
    학습자 (Learner)교재 학습, 퀴즈 풀이, 프롬프트 실습, 진도 관리, 인증서 취득
    관리자 (Admin)교재 생성/관리, 학습 분석, 수강생 관리, 트랙 구성, 목표 설정
    전문가 (Expert)교재 리뷰, 멘토링, 학습 가이드
    팀 리더팀원 학습 현황 파악, 그룹 목표 관리

    콘텐츠 계층 구조

    DAKER LMS는 3-Tier 계층 구조로 학습 콘텐츠를 구성한다.

    3.1 계층도

    학습 트랙 (Track)
  └─ 코스 (Course) — 교재 모음
       └─ 교재 (Material) — 단일 학습 단위
            └─ 스테이지 (Stage) — 학습 활동 단위 (6종)

    3.2 각 계층 상세

    트랙 (Track)

    속성설명
    정의하나의 학습 목표를 달성하기 위한 코스·교재의 순서 있는 묶음
    유형industry (산업별), skill (기술별), custom (커스텀)
    진행 모드순차 강제, 순차 선택, 자유
    연계Raid/해커톤과 연계하여 참가자 사전 학습 경로로 활용
    생성 권한관리자 또는 크리에이터 권한 보유자

    코스 (Course)

    속성설명
    정의교재(Material)의 논리적 그룹. 하나의 주제 또는 모듈 단위
    진행 모드sequential_forced (순차 강제), sequential_optional (순차 선택), free (자유)
    상태draft, published, archived
    필수/선택트랙 내에서 필수 또는 선택 코스로 지정 가능

    교재 (Material)

    속성설명
    정의학습의 기본 단위. 여러 스테이지로 구성
    난이도beginner, intermediate, advanced
    상태draft → published → archived
    메타데이터제목, 설명, 썸네일, 카테고리, 태그, 예상 소요시간, 학습 목표
    조회수자동 집계
    AI 생성Gemini 기반 자동 생성 지원

    스테이지 (Stage)

    속성설명
    정의교재 내 개별 학습 활동. 화면 하나에 대응
    유형6종 (content, quiz, prompt, flashcard, pdf, branching-lesson)
    순서orderIndex로 정렬
    슬러그URL-friendly 식별자
    팁 콘텐츠학습 중 도움말로 표시되는 부가 설명
    4

    스테이지 유형 및 플러그인 시스템

    4.1 6종 스테이지 유형 상세

    4.1.1 콘텐츠 (Content)

    항목설명
    유형content
    목적마크다운 기반 텍스트 학습 콘텐츠
    채점비채점
    데이터content 필드 (마크다운 텍스트)
    AI 생성지원 (500~1500자 권장)
    뷰어LearningMarkdownRenderer — 코드 하이라이팅, 테이블, 이미지 등 지원

    4.1.2 퀴즈 (Quiz)

    항목설명
    유형quiz
    목적4지선다형 객관식 문제
    채점채점 가능 (80% 이상 통과)
    데이터 형식{ question, options: [{id, text}], correctAnswer: optionId, explanations, hasExplanation }
    시도 기록learningQuizAttempts 테이블에 답변, 점수, 통과 여부 저장
    AI 생성지원
    뷰어LearningQuizViewer — 정답 확인, 해설 표시, 재시도

    4.1.3 프롬프트 실습 (Prompt)

    항목설명
    유형prompt
    목적AI 프롬프트 작성 실습. 사용자가 작성한 프롬프트를 AI가 평가
    채점채점 가능 (유사도 점수 0~100)
    데이터 형식{ instruction, sampleAnswer, hasSampleAnswer }
    평가 항목의도(intent), 구조(structure), 명확성(clarity) 3축 평가
    시도 기록learningPromptAttempts 테이블에 사용자 프롬프트, AI 응답, 유사도, 피드백 저장
    뷰어LearningPromptViewer — 프롬프트 입력, AI 평가 결과 표시

    4.1.4 플래시카드 (Flashcard)

    항목설명
    유형flashcard
    목적앞/뒤 카드 기반 암기 학습
    채점채점 가능 (Known/Unknown 비율)
    데이터 형식{ cards: [{id, front, back, difficulty}], settings: {shuffleOnLoad, showProgress, passingThreshold} }
    난이도카드별 easy, medium, hard
    시도 기록learningFlashcardAttempts 테이블에 카드별 결과, 소요 시간 저장
    뷰어LearningFlashcardViewer — 카드 플립 애니메이션, 진도 표시

    4.1.5 PDF

    항목설명
    유형pdf
    목적PDF 문서 뷰어 (외부 자료 참조)
    채점비채점
    데이터PDF 파일 URL
    AI 생성미지원 (외부 파일 업로드)
    뷰어LearningPdfViewer — 내장 PDF 리더

    4.1.6 분기 학습 (Branching Lesson)

    항목설명
    유형branching-lesson
    목적시나리오 기반 트리 구조 학습. 학습자의 선택에 따라 경로 분기
    채점비채점
    데이터 구조노드 트리 (각 노드에 콘텐츠 + 선택지 + 다음 노드 ID)
    뷰어LearningBranchingLessonViewer — 대화형 선택지 UI

    4.2 플러그인 확장 방법

    새로운 스테이지 유형 추가 시:

    1. server/learning/plugins/{type}.ts     — StagePlugin 인터페이스 구현
2. client/src/learning/plugins/{type}.ts  — ClientStagePlugin 어댑터
3. server/learning/plugins/registry.ts    — import + Map 등록 (1줄)
4. client/src/learning/plugins/client-registry.ts — import + Map 등록 (1줄)
5. shared/schema/learning.ts의 STAGE_TYPES에 타입 추가

    5.2 진도 추적

    상태설명
    not_started교재를 열지 않은 상태
    in_progress하나 이상의 스테이지를 진행 중
    completed모든 스테이지 완료

    진도 데이터:

    • completedStages: 완료한 스테이지 ID 배열 (JSON)
    • learningTimeSeconds: 총 학습 시간 (초)
    • stageTimeMap: 스테이지별 학습 시간 (JSON: stageId → seconds)
    • stageSummaries: 스테이지별 학습 요약 (JSON: stageId → summary)
    • completionCount: 완료 횟수 (반복 학습 추적)

    5.3 xAPI 이벤트 스트림

    모든 학습 행동은 xAPI 규격의 이벤트로 기록된다.

    Verb설명예시
    viewed콘텐츠 조회교재 페이지 열람
    started학습 시작교재/트랙 학습 시작
    attempted시도퀴즈 풀이 시작
    answered응답퀴즈 답 제출
    completed완료교재/스테이지 완료
    passed통과퀴즈 80% 이상
    failed미통과퀴즈 80% 미만
    submitted제출프롬프트 답변 제출
    commented코멘트학습 노트 작성
    branched분기분기 학습 선택
    abandoned이탈학습 도중 이탈
    resumed재개이탈 후 학습 재개

    Object Type: material, stage, question, assignment, track

    AI 교재 생성 파이프라인

    6.1 개요

    DAKER LMS의 핵심 차별 기능으로, AI가 교재를 자동 생성한다.

    6.2 파이프라인 구조

    [관리자: 주제/키워드 입력]
            │
            ▼
┌─────────────────────────────────┐
│  Step 1: 커리큘럼 생성 (Gemini) │
│  주제 분석 → 학습 목표 도출     │
│  → 스테이지 구성 계획           │

               │
               ▼
┌─────────────────────────────────┐
│  Step 2: 상호 검수 (GPT)       │
│  Gemini 결과를 GPT가 리뷰      │
│  → 개선점 피드백               │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  Step 3: 콘텐츠 생성 (Gemini)  │
│  각 스테이지별 콘텐츠 생성      │

    6.3 AI 생성 기능 목록

    기능설명스트리밍
    단일 교재 생성주제/키워드로 전체 교재 생성SSE
    일괄 교재 생성트랙 내 여러 교재를 한 번에 생성SSE
    커리큘럼 AI 생성트랙 전체 커리큘럼 자동 설계SSE
    커리큘럼 AI 보완기존 커리큘럼 개선 제안SSE
    프롬프트 스펙 파싱자연어 명세서로부터 교재 구조 생성SSE
    학습 AI 어시스턴트학습 중 질의응답 챗봇SSE

    6.4 플러그인 연동

    각 서버 플러그인의 aiPromptFragment 속성에 해당 스테이지 유형의 데이터 형식이 한국어로 명시되어 있으며, AI 교재 파이프라인이 이를 자동으로 프롬프트에 포함한다. PDF처럼 AI 생성 대상이 아닌 유형은 자동 제외된다.

    6.5 일일 토큰 제한

    AI 생성에는 일일 토큰 한도가 적용되며, aiGenerationLogs 테이블에 입력/출력 토큰, 사용 여부, 피드백이 기록된다.

Step 2: 커리큘럼
  ─ 스테이지 구성 (수동 또는 AI 자동 생성)
  ─ 스테이지 순서 설정

Step 3: 콘텐츠
  ─ 각 스테이지별 콘텐츠 입력/편집
  ─ 퀴즈, 프롬프트, 플래시카드 데이터 입력

Step 4: 미리보기 & 발행
  ─ 전체 교재 미리보기
  ─ 발행 (draft → published)
