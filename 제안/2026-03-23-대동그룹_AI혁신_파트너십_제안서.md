## II. 부품 수요 예측 경진대회 설계

4년치 Daily 판매 이력 기반, 전국 대리점 부품 재고 최적화 AI 모델 개발

### 대회 핵심 정보

| 항목 | 내용 |
|------|------|
| 대회명 | 대동 트랙터 부품 수요 예측 챌린지 |
| 주제 | 부품별 월간/주간 수요량 예측 |
| 데이터 | 4년치 Daily 부품 판매 이력 |
| 평가 지표 | RMSE / MAPE / 비즈니스 KPI |
| 참가 형태 | 개인 또는 팀 (최대 5인) |
| 기간 | 4~6주 (예선 + 본선) |

### 기대 효과

**재고 비용 절감**
- 과잉 재고 **15~30% 절감** 가능
- 품절률 최소화로 대리점 서비스 향상

**다양한 모델 확보**
- 수백 개 팀의 접근법 중 **Top N 모델 앙상블**
- 단일 모델 대비 정확도 향상

**AI 인재 발굴**
- 우수 참가자 **채용 연계** 가능
- 대동 AI Lab 인재 파이프라인 구축

## II. 데이터셋 정의 및 운영 절차

### 필요 데이터셋 구성

| 데이터셋 | 컬럼 예시 | 비고 |
|----------|-----------|------|
| 부품 판매 이력 | 날짜, 부품코드, 대리점ID, 판매수량 | 4년 Daily |
| 부품 마스터 | 부품코드, 카테고리, 단가, 리드타임 | 정적 데이터 |
| 대리점 정보 | 대리점ID, 지역, 규모, 장비보유수 | 익명화 |
| 계절/외부 변수 | 농번기, 기온, 강수량, 공휴일 | 공개 데이터 |
| 프로모션 이력 | 캠페인 기간, 할인율, 대상 부품 | 선택 제공 |

### 데이터 보안 원칙

- 대리점/고객 정보 완전 익명화
- NDA 체결 후 데이터 제공
- 대회 종료 후 참가자 데이터 삭제

### 운영 4단계 프로세스

**01. 사전 준비 (2~3주)**
- 데이터 정제 · 익명화 · EDA 문서 작성
- 평가 지표 확정 · 베이스라인 모델 수립

**02. 예선 (3~4주)**
- 온라인 리더보드 경쟁
- Public/Private 데이터셋 분리 · 중간 멘토링

**03. 본선 (1~2주)**
- Top 10팀 발표 심사
- 비즈니스 적용 가능성 + 모델 성능 종합 평가

**04. 사후 (2주)**
- 수상 모델 기술 이전 · 결과 보고서
- 우수 인재 채용 연계 · 사내 적용 컨설팅

## V. 왜 대동에 바이브코딩 대회가 필요한가

AI가 코드를 쓰는 시대, 비개발자도 소프트웨어를 만들 수 있습니다

### 3가지 이유

**01. AX 전환 가속**
- 대동의 WI(Work Innovation) 전략과 **완벽 부합**
- 임직원이 직접 업무 도구를 AI로 만들어 **자체 혁신 역량**을 내재화합니다

**02. 진입장벽 해소**
- 바이브코딩은 **코딩 경험이 없어도** 참여 가능
- 제조, 영업, 경영지원 등 **전 부서 임직원**이 경쟁에 참여합니다

**03. 실전 결과물 확보**
- 대회 결과물이 **실제 업무 도구**로 활용 가능
- 재고 대시보드, 품질 분석 앱 등 **즉시 적용** 가능한 프로토타입 확보

| 바이브코딩 사용자 중 비개발자 비율 | **63%** |
| Fortune 500 도입률 | **87%** |
| AI 활용 시 작업 속도 향상 | **55%** |
