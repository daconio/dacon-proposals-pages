## IV. 예선 구성 및 평가 방법

### 온라인 예선 운영

데이콘 플랫폼(dacon.io)에서 **리더보드 기반 자동 채점** 방식으로 운영

#### 예선 진행 방식

| 항목 | 내용 |
|------|------|
| 플랫폼 | dacon.io (데이콘 공식 플랫폼) |
| 기간 | 약 6주 (7월 초~8월 중순) |
| 데이터 제공 | Train Data + Test Data 제공 |
| 제출 방식 | 예측 결과 파일(CSV) 제출 |
| 채점 | **자동 채점** — 실시간 리더보드 반영 |
| 제출 횟수 | 1일 최대 3~5회 (조정 가능) |

#### 제출 데이터 평가 방법

| 구분 | 내용 |
|------|------|
| 평가 지표 | 과제 특성에 따라 결정 (예: RMSE, MAE, F1-Score, AUC 등) |
| Public 리더보드 | 예선 기간 중 **Test Data의 일부(30~40%)**로 실시간 순위 공개 |
| Private 리더보드 | 예선 종료 후 **Test Data의 나머지(60~70%)**로 최종 순위 확정 |
| 최종 선발 기준 | **Private 리더보드** 순위 기준 |

> Public/Private 분리는 과적합(Overfitting) 방지 및 일반화 성능 검증을 위한 표준 방식입니다

#### 부정행위 방지

| 항목 | 방법 |
|------|------|
| 코드 제출 의무 | 예선 상위팀 **소스코드 제출** 필수 |
| 코드 재현 검증 | 제출 코드로 결과 **재현 가능 여부** 확인 |
| 유사도 검사 | 팀 간 코드 유사도 자동 분석 |
| 외부 데이터 제한 | 허용 외부 데이터 사전 명시 |
