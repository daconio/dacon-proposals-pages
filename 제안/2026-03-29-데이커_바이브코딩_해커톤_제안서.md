## 1. 바이브코딩이란 — AI 시대의 새로운 경쟁 방식

### 정의

**바이브코딩(Vibe Coding)**은 AI와 자연어로 대화하며 소프트웨어를 만드는 새로운 개발 방식입니다. 2025년 2월, Tesla AI 수장 출신 Andrej Karpathy가 처음 제안한 개념으로, 개발자가 코드를 한 줄씩 작성하는 대신 AI에게 의도와 맥락을 전달하여 프로토타입부터 완성품까지 빠르게 구현하는 접근법을 의미합니다.

> *"There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."*
> — Andrej Karpathy, 2025.02

### 기존 코딩과의 3가지 핵심 차이

| 구분 | 기존 코딩 | 바이브코딩 |
|------|-----------|-----------|
| **핵심 역량** | 프로그래밍 언어 문법, 알고리즘 구현력 | 문제 정의, 기획력, AI 협업 능력 |
| **개발 방식** | 코드를 직접 작성하고 디버깅 | AI에게 의도를 전달하고 결과를 검증 |
| **참여 대상** | 개발자 중심 | 기획자, 디자이너, 도메인 전문가 등 다양한 직군 |

#### 차별점 1: 문제 해결 중심

바이브코딩에서 가장 중요한 역량은 코드 문법이 아닌 **문제를 정의하고 해결하는 능력**입니다. "무엇을 만들 것인가"에 대한 명확한 비전과, 이를 AI에게 효과적으로 전달하는 커뮤니케이션 역량이 핵심입니다.

#### 차별점 2: AI 활용 역량

Claude Code, Cursor, GitHub Copilot 등 AI 코딩 도구를 활용하여 **아이디어를 빠르게 프로토타입으로 전환**합니다. AI가 생성한 코드를 검증하고, 반복적으로 개선하며, 최종 결과물의 품질을 관리하는 것이 개발자의 새로운 역할입니다.

#### 차별점 3: 진입 장벽 해소

코딩 경험이 없는 비전공자도 참여할 수 있어 **더 다양한 참여자를 확보**할 수 있습니다. 경영학과, 디자인학과, 인문학과 등 다양한 전공의 학생들이 직접 서비스를 만들어볼 수 있는 시대가 열렸습니다.

### 왜 지금인가

- **AI 코딩 도구의 성숙**: Claude Code, Cursor, Windsurf 등 생산성 도구가 실무 수준에 도달
- **글로벌 트렌드**: 미국, 유럽 주요 기업들이 바이브코딩 해커톤을 사내 혁신 프로그램으로 도입
- **정부 정책 방향**: 디지털 인재 양성, AI 활용 역량 강화 등 정책 기조와 부합
- **기업 수요**: "AI를 활용할 줄 아는 인재"에 대한 채용 시장의 수요 폭증

## 2. 왜 단국대학교에 바이브코딩 해커톤인가

### 단국대 SW중심대학과의 시너지

단국대학교는 2024년 SW중심대학으로 선정되어 6년간 110억원을 지원받으며, 전교생 대상 SW·AI 필수 교양 교육을 운영하고 있습니다 (연간 3,000명+ 이수). 바이브코딩 해커톤은 이러한 교육 성과를 **실전 프로젝트로 실증**하는 최적의 프로그램입니다.

- SW중심대학 교양을 이수한 비전공자도 AI 도구를 활용하여 실제 서비스를 만들 수 있음을 증명
- DACON 알고리즘 경진대회와 함께 **2개 대회 동시 개최**로 SW중심대학 성과 지표에 다각도 기여
- 비전공자 SW교육 성과, 팀 프로젝트, AI 도구 활용 등 핵심 성과 지표에 직접 매핑

### 기존 해커톤 vs 바이브코딩 해커톤

| 비교 항목 | 기존 해커톤 | 바이브코딩 해커톤 |
|-----------|-----------|-----------------|
| 참가 대상 | 개발자 중심 (CS 전공자) | 단국대 전교생 참여 가능 (비전공자 포함, SW중심대학 교양 이수자 포함) |
| 진입 장벽 | 높음 (프로그래밍 필수) | 낮음 (AI 도구 활용) |
| 결과물 수준 | 불완전한 프로토타입이 다수 | 실사용 가능한 MVP 수준 달성률 높음 |
| 평가 기준 | 코드 품질, 기술 난이도 중심 | 문제 해결력, 사용자 경험, 사업성 종합 평가 |
| 팀 구성 | 개발자 위주 동질적 팀 | 다양한 역할 기반 이질적 팀 |
| 완주율 | 40~50% | 70% 이상 |
| 참가 규모 | 소규모 (50~100명) | 대규모 (300~1,000명+) |

### 주최 기관을 위한 기대 효과

#### 참가율 향상

비전공자까지 참여 대상이 확대되어 기존 해커톤 대비 **3배 이상의 참가자**를 확보할 수 있습니다. 실제 데이콘이 운영한 SeSAC 바이브코딩 해커톤에서는 923명이 참가하여 역대 최대 규모를 기록했습니다.

#### 완주율 상승

AI 도구의 도움으로 기술적 벽에 막혀 중도 포기하는 비율이 크게 줄어 **70% 이상의 완주율**을 달성합니다. 참가자 입장에서 "결과물을 낼 수 있다"는 확신이 지속적인 참여 동기를 부여합니다.

#### 결과물 품질 향상

기획자의 비즈니스 감각, 디자이너의 UX 역량, 도메인 전문가의 현장 지식이 결합되어 **실제 사업화 가능한 수준의 결과물**이 도출됩니다. 기술만 뛰어난 결과물이 아닌, 시장 적합성이 높은 솔루션을 기대할 수 있습니다.

#### 커뮤니티 형성

대회 이후에도 참가자 간 네트워크가 유지되어 **지속적인 혁신 생태계**가 조성됩니다. 주최 기관은 AI 혁신 허브로서의 브랜드 포지셔닝을 확보할 수 있습니다.

### 참가자를 위한 기대 효과

#### 실무형 경험

단순한 코딩 연습이 아닌, **실제 문제를 해결하는 프로젝트 경험**을 쌓을 수 있습니다. AI 도구를 활용한 개발 프로세스는 현업에서 즉시 적용 가능한 역량으로 이어집니다.

#### 역할 기반 성장

기획, 개발, 디자인, 프로젝트 관리 등 **자신의 강점에 맞는 역할**로 참여하여 해당 분야의 역량을 심화할 수 있습니다. 특히 비개발 직군은 AI 시대에 필요한 새로운 스킬셋을 체득하게 됩니다.

#### 포트폴리오

실제 작동하는 서비스를 만들어 **취업, 이직, 사내 평가에 활용할 수 있는 포트폴리오**를 확보합니다. DAKER 플랫폼의 갤러리 기능을 통해 결과물이 영구 보존되며 공개됩니다.

#### 네트워킹

다양한 배경의 참가자와 협업하며 **전문 분야를 넘나드는 인적 네트워크**를 구축합니다. 멘토, 심사위원과의 교류를 통해 업계 인사이트를 얻을 수 있습니다.

## 3. 대회 유형 및 트랙 구성

단국대학교 SW중심대학의 교육 목표와 학생 특성에 맞춘 4개 트랙을 운영합니다. 참가 대상은 **단국대 전교생 (비전공자 포함)**이며, SW중심대학 교양 이수자를 포함합니다.

### Track A: 캠퍼스 라이프

단국대 학생들의 학교생활을 개선하는 AI 서비스를 개발하는 트랙

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 학사 도우미, 수강 추천, 캠퍼스 편의 서비스 |
| **대상** | 단국대 전교생 (비전공자 포함) |
| **방식** | 팀 단위(3~5명, 학과 혼합 권장) 바이브코딩으로 MVP 개발 |
| **평가 기준** | 실용성 25%, 혁신성 25%, 사용자 경험 20%, 완성도 20%, 발표 10% |
| **결과물** | 실제 작동하는 웹/앱 서비스 (배포 URL 제출) |
| **제출물** | 기획서(PDF) + MVP URL + 소스코드(GitHub) + 발표자료 |

### Track B: 지역 연계

천안/용인 지역 사회와 연계한 AI 기반 서비스를 개발하는 트랙

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 천안/용인 지역 특화 서비스 (교통, 문화, 소상공인, 관광 등) |
| **대상** | 단국대 전교생 (비전공자 포함) |
| **방식** | 지역 문제 발굴 → AI 솔루션 기획 → MVP 개발 |
| **평가 기준** | 지역 임팩트 30%, 실현 가능성 25%, 혁신성 20%, 완성도 15%, 발표 10% |
| **결과물** | 지역 문제 해결 AI 서비스 프로토타입 |
| **제출물** | 문제 정의서 + 솔루션 기획서 + MVP + 발표자료 |

### Track C: 산업 특화

자동차, 헬스케어, 반도체 등 단국대 강점 산업 분야 AI 솔루션을 개발하는 트랙 (IDC SW아카데미 연계)

| 항목 | 내용 |
|------|------|
| **핵심 주제** | 자동차/헬스케어/반도체 산업 AI 솔루션 |
| **대상** | 단국대 전교생 (관련 학과 및 IDC SW아카데미 수료생 우대) |
| **방식** | 산업 과제 기반 팀 단위 AI 솔루션 개발 |
| **평가 기준** | 산업 적용 가능성 30%, 혁신성 20%, 기술 완성도 20%, 실용성 15%, 발표 15% |
| **결과물** | 산업 특화 AI 솔루션 프로토타입 |
| **제출물** | 솔루션 기획서 + 프로토타입 + 적용 시나리오 + 발표자료 |

### Track D: 자유 주제

사회 문제 해결 SW를 자유롭게 기획하고 개발하는 트랙

| 항목 | 내용 |
|------|------|
| **핵심 주제** | AI 활용 사회 문제 해결 소프트웨어 (주제 제한 없음) |
| **대상** | 단국대 전교생 (비전공자 포함) |
| **방식** | 사전 교육(2~4시간) + 바이브코딩 개발(온라인 2주) |
| **평가 기준** | 창의성 25%, 사회적 임팩트 25%, 완성도 20%, 팀워크 20%, 발표 10% |
| **결과물** | AI 기반 서비스 프로토타입 |
| **제출물** | 기획서 + MVP + 소스코드(GitHub) + 발표자료 |

## 4. DAKER 플랫폼 — 해커톤의 모든 것

DAKER(daker.ai)는 데이콘이 7년간의 AI 대회 운영 노하우를 집약하여 구축한 **올인원 AIX(AI Experience) 플랫폼**입니다. 대회 기획부터 참가 접수, 팀 구성, 협업, 제출, 심사, 아카이브까지 해커톤의 전체 라이프사이클을 하나의 플랫폼에서 운영합니다.

### 대회 등록 및 참가 접수

- 대회 목록 페이지에서 상태별 필터링 (모집 중 / 진행 중 / 종료)
- 대회별 상세 안내 페이지 자동 생성
- 개인정보처리동의서 전자 서명 수집
- 다단계 동의 체크 (결과물 활용, 초상권, 개인정보 등)
- 참가 현황 실시간 대시보드 (주최측 어드민)

### 5대 핵심 탭

모든 대회는 아래 5가지 탭으로 통일된 구조를 제공합니다.

| 탭 | 기능 |
|----|------|
| **대회안내** | 대회 개요, 일정, 규칙, FAQ, 상금 정보 |
| **제출** | 기획서, 프로토타입, 발표자료 등 다단계 제출 관리 |
| **리더보드** | 실시간 순위 및 심사 결과 공개 |
| **갤러리** | 타 팀 결과물 조회 (공개일 이후), 좋아요/댓글 |
| **게시판** | 공지사항, Q&A, 멘토 질의응답 |

### 원정대(Raid) 팀 구성 시스템

DAKER의 팀 구성 시스템은 해커톤에 특화된 기능을 제공합니다.

- **팀명 및 Slug 자동 생성**: 팀 생성 시 고유 식별자 자동 부여
- **팀 목표 설정**: 팀의 비전과 목표를 명시하여 방향성 공유
- **기술 스택 태그**: 사용할 AI 도구 및 기술 스택을 태그로 관리
- **멤버 슬롯 (역할별 배정)**
  - Leader: 팀 리더, 프로젝트 총괄
  - Frontend: UI/UX 구현 담당
  - Backend: 서버/API 개발 담당
  - AI/ML: AI 모델 및 프롬프트 엔지니어링
  - Design: 디자인 및 사용자 경험 설계
- **팀원 초대 및 합류**: 이메일 초대, 초대 링크 공유, Auto-Sync 연동

### 작전실(War Room) — 실시간 협업 공간

팀별로 제공되는 전용 협업 공간으로, 해커톤 기간 동안 모든 작업이 이곳에서 이루어집니다.

- **주최자 맞춤형 100% 미러링**: 주최측이 설계한 제출 양식과 과제 구조가 작전실에 그대로 반영
- **Yjs WebSocket 기반 실시간 공동 편집**: Google Docs 수준의 동시 편집 경험 제공
- **멀티 커서 트래킹**: 팀원들이 동시에 작업하는 위치를 실시간으로 확인
- **상태 배지**: 각 제출물의 진행 상태를 시각적으로 표시
  - 미작성 → 작성 중 → 최종제출 완료
- **GitHub 연동**: 소스코드 저장소와 자동 연결, 커밋 이력 추적

### 전용 게시판 및 커뮤니티

- 대회별 전용 게시판 (공지사항, Q&A, 자유게시판)
- 멘토 질의응답 전용 채널
- 실시간 알림 (공지, 답변, 일정 변경 등)

### 무손실 데이터 저장소 (Archive)

- 대회 종료 후에도 모든 제출물, 협업 기록, 피드백이 **영구 보존**
- 참가자 개인 프로필에 업적 배지(Achievement Badge) 부여
- 결과물 갤러리를 통한 외부 공개 및 포트폴리오 활용

### 테마 모드

사용자 취향에 맞는 3가지 테마 지원

- **라이트 모드**: 밝은 환경에서의 가독성 최적화
- **다크 모드**: 장시간 작업 시 눈의 피로 경감
- **브루탈 모드**: 몰입감 높은 해커톤 전용 인터페이스

## 5. 대회 운영 프로세스 (총 8주)

```
[W1: 기획/준비] → [W2: 홍보/모집] → [W3~W4: 팀 빌딩+사전 교육] → [W5~W6: 바이브코딩 개발] → [W7: 제출+심사] → [W8: 데모데이+시상]
```

### W1: 기획 및 준비

단국대학교와 데이콘이 협력하여 대회의 전체 설계를 완성합니다.

| 단계 | 세부 내용 | 담당 |
|------|----------|------|
| 주제 선정 | 대회 목적에 부합하는 주제 및 트랙 확정 | 단국대학교 + 데이콘 |
| 과제 설계 | 평가 기준, 제출물 양식, 심사 체계 수립 | 단국대학교 + 데이콘 |
| 일정 수립 | 8주 전체 타임라인 확정 | 단국대학교 + 데이콘 |
| 플랫폼 세팅 | DAKER 대회 페이지 구축, 제출 양식 구성 | 데이콘 |
| 홍보 계획 | 학내 홍보 전략 및 콘텐츠 제작 | 단국대학교 + 데이콘 |

### W2: 홍보 및 모집 (단국대 내부)

단국대학교 내부 채널을 통해 참가자를 모집합니다. (비공개 대회)

| 단계 | 세부 내용 | 담당 |
|------|----------|------|
| 홍보 페이지 | 대회 소개 랜딩 페이지 공개 | 데이콘 |
| 학내 홍보 | 학과 게시판, LMS, 학생회, SW중심대학 채널 활용 | 단국대학교 |
| 참가 접수 | 개인/팀 단위 접수, 개인정보 동의 수집 | DAKER 플랫폼 |
| 대상 안내 | 전교생 대상 (비전공자 포함, SW중심대학 교양 이수자 포함) | 단국대학교 |

### W3~W4: 팀 빌딩 + 사전 교육

팀을 구성하고, 바이브코딩 사전 교육을 진행합니다.

| 단계 | 세부 내용 | 담당 |
|------|----------|------|
| 팀 구성 | 3~5인 팀 구성 (학과 혼합 권장), 자유 구성 또는 자동 매칭 | DAKER 플랫폼 |
| 사전 교육 | 바이브코딩 도구 사용법, 프롬프트 엔지니어링, MVP 개발 실습 | 데이콘 |
| 멘토 배정 | 교수 멘토 및 기술 멘토 팀별 배정 | 단국대학교 + 데이콘 |
| OT | 참가자 오리엔테이션, 규칙 설명, 트랙별 과제 안내 | 데이콘 |

### W5~W6: 바이브코딩 개발 (온라인 2주)

참가자들이 DAKER 플랫폼에서 온라인으로 과제를 수행합니다.

| 단계 | 세부 내용 | 담당 |
|------|----------|------|
| 과제 수행 | 작전실에서 팀별 협업, 기획서 및 MVP 개발 | 참가팀 |
| 중간 점검 | 진행 현황 모니터링, 중간 발표(선택) | 데이콘 + 단국대학교 |
| 멘토링 | 기술/비즈니스 멘토 배정, Q&A 게시판 운영 | 멘토단 (교수 + 외부) |
| 기술 지원 | AI 도구 사용법, 배포 가이드 등 기술 서포트 | 데이콘 |

### W7: 제출 마감 + 심사

최종 제출 및 심사위원단의 평가가 진행됩니다.

| 단계 | 세부 내용 | 담당 |
|------|----------|------|
| 최종 제출 | 기획서 + MVP + 발표자료 통합 제출 | 참가팀 |
| 서류 심사 | 기획서 및 결과물 1차 평가 | 심사위원단 |
| MVP 심사 | 실제 서비스 시연 및 기술 완성도 평가 | 심사위원단 |

### W8: 데모데이 + 시상 (교내)

상위 팀 대상 교내 데모데이 및 시상식을 개최합니다.

| 단계 | 세부 내용 | 담당 |
|------|----------|------|
| 최종 발표 | 상위 팀 대상 교내 데모데이 발표 | 심사위원단 (단국대 교수 + 외부) |
| 리더보드 공개 | 최종 순위 및 심사 결과 공개 | DAKER 플랫폼 |
| 시상식 | 수상팀 발표, 상금 지급, 기념 촬영 | 단국대학교 |
| 갤러리 오픈 | 전체 팀 결과물 공개, 커뮤니티 피드백 | DAKER 플랫폼 |

### 운영 방식

| 항목 | 내용 |
|------|------|
| 진행 방식 | 온라인 개발(W5~W6) + 교내 데모데이(W8) |
| 총 기간 | 8주 |
| 규모 | 단국대 전교생 대상 비공개 대회 |
| 장점 | 온라인 개발로 접근성 확보 + 교내 데모데이로 몰입감 제공 |

## 6. 팀 중심 참여 구조

바이브코딩 해커톤의 핵심은 **다양한 학과의 팀원들이 협업**하는 것입니다. DAKER 플랫폼은 팀 구성부터 협업, 제출까지 전 과정을 지원합니다.

### 팀 구성 방식

- **팀 규모**: 3~5인 (학과 혼합 권장)
- **구성 원칙**: 비전공자와 전공자가 함께 팀을 이루어 다양한 관점을 결합

| 방식 | 설명 | 적합 상황 |
|------|------|----------|
| **자유 구성** | 참가자가 직접 팀을 구성하여 참가 | 동아리, 스터디 그룹 등 기존 네트워크 활용 |
| **자동 매칭** | 희망 역할, 학과, 관심 분야 기반 매칭 | 개인 참가자 비율이 높은 경우 |
| **혼합 방식** | 팀 구성 후 부족 역할을 매칭으로 보충 | 학과 간 교류 촉진 |

### 역할 배정 체계

| 역할 | 주요 업무 | 바이브코딩에서의 역할 |
|------|----------|-------------------|
| **팀장 (Leader)** | 프로젝트 총괄, 일정 관리, 의사결정 | AI와의 대화 방향 설정, 결과물 품질 관리 |
| **기획 (Planner)** | 서비스 기획, 사용자 조사, 비즈니스 모델 | 문제 정의, 요구사항 명세, 기획서 작성 |
| **개발 (Developer)** | 프론트엔드/백엔드 구현, 배포 | AI 코딩 도구 활용, 코드 검증 및 디버깅 |
| **디자인 (Designer)** | UI/UX 설계, 디자인 시스템 구축 | AI 디자인 도구 활용, 사용자 경험 최적화 |
| **AI/ML** | AI 모델 연동, 프롬프트 엔지니어링 | LLM 활용 전략 수립, API 연동 |

### 협업 수행 환경

- **공유 대시보드**: 팀 진행 현황, 마일스톤, 체크리스트 실시간 공유
- **실시간 공동 편집**: 작전실(War Room)에서 기획서, 코드, 발표자료 동시 작업
- **상태 추적**: 각 제출물의 작성 현황을 팀원 전원이 확인
- **GitHub 연동**: 코드 버전 관리 및 기여도 추적

### 핵심 지표

| 지표 | 수치 | 비고 |
|------|------|------|
| 참가자 확보율 | 기존 대비 **3배** | 비전공자 참여 확대 효과 |
| 평균 완주율 | **85%** | AI 도구 지원으로 기술적 장벽 해소 |
