# DAKER AI 멘토링 — 개념·프로세스·UI 개요

> 해커톤 참가팀에게 CP(체크포인트) 기반 구조화 멘토링과 자유 채팅을 제공하는 AI 멘토링 시스템의 전반 개념, 운영 프로세스, UI를 한 번에 정리합니다.

---

## 1. Cover — AI 멘토링이란

**한 줄 정의**: 해커톤 전 기간 동안 팀에게 필요한 순간에, 정해진 질문 흐름으로 AI 멘토가 붙어 완주를 돕는 시스템.

- 적용 위치: 팀 워룸(참가자) + 해커톤 관리자 콘솔(운영자)
- 4개 Checkpoint + 자유 채팅 = 총 5개 세션 타입
- 기술 스택: Gemini (AIProvider 추상화), SSE 스트리밍, Drizzle/PostgreSQL

---

## 2. 왜 만들었나 — 4가지 문제 동시 해결

1. **B2B 교육 상품화** — DAKER 플랫폼의 부가가치 창출 (최우선)
2. **멘토링 품질 균등화** — 사람 멘토 역량 편차 해소
3. **참가자 사각지대 발견** — 구조화된 질문으로 스코프·리스크 조기 노출
4. **멘토 부족 해소** — 24/7 AI 멘토링 제공

## 4. 5개 세션 타입 — CP 4개 + Free Chat

| 타입 | 라벨 | AI 역할 | 핵심 지시 |
|------|------|---------|-----------|
| `team_setup` | 팀 셋업 | Team Coach | 역할 분배·일정 최적화. 프로필 확인형 첫 질문 |
| `idea_review` | 아이디어 리뷰 | Idea Mentor | 리프레이밍. "실제로 만들고 있는 건 X" |
| `mid_check` | 중간 점검 | Tech Mentor | 남은 기간 대비 스코프 컷 대담하게 제안 |
| `final_review` | 최종 리뷰 | Submit Reviewer | 루브릭 체크리스트 · quick-win 집중 |
| `free_chat` | 자유 채팅 | General Mentor | 힌트 기반. 정답 직접 제공 금지, 코드 50줄 이하 |

각 CP는 **강제 질문 흐름**(QuestionItem 배열) + **리포트 자동 생성**으로 구성.

## 5. 진입 프로세스 — 동의부터 세션까지

```
[AI 멘토링 탭 진입]
      │
      ▼
동의 상태 확인  ── 미동의 ──▶ MentoringConsentDialog
  (useMentoringConsentStatus)         │
      │ 동의 완료                     ▼
      ▼                        서버 403 CONSENT_REQUIRED
CP 카드 클릭 OR 자유 채팅             │
      │                               ▼
      ▼                        동의 저장 → pendingAction 재개
POST /api/mentoring/sessions
      │
      ├─ 409 Conflict(진행중) ──▶ 기존 세션 이어가기
      │
      ▼
MentoringChat 뷰 진입 → SSE로 첫 질문 수신
```

핵심: **동의 게이트**는 클라이언트(`hasConsented`)와 서버(403 코드) 양쪽에서 강제.

## 6. CP 세션 실행 플로우 (SSE)

```
사용자 답변 입력 (≤2000자)
   │
   ▼
POST /api/mentoring/sessions/:id/messages
   │
   ▼ (서버)
1. 일일 토큰 한도 확인 (429 체크)
2. user 메시지 저장
3. 컨텍스트 구성: 시스템 프롬프트 + 이전 CP 요약 + 최근 10턴
4. AIProvider.chatStream() 호출
5. SSE chunk: {"chunk": "..."}
6. 완료: {"done": true, "tokenCount": N}
7. assistant 메시지 저장 + 세션 tokenUsage 업데이트
8. 모든 질문 완료 시 → mentoring_reports 자동 생성
```

미지원 클라이언트는 `?stream=false`로 JSON 응답 받기 가능.

## 7. CheckpointCard — 상태 3종

```
┌─ Completed ─────────────┐   ┌─ Available ─────────────┐   ┌─ Locked ────────────────┐
│ ✅ CP1. 팀 셋업           │   │ 🔵 CP2. 아이디어 리뷰   │   │ 🔒 CP3. 중간 점검        │
│ 2026-04-13 14:30        │   │ "이전 CP 완료"          │   │ 이전 단계 완료 필요       │
│ ★★★★☆ 4.0              │   │                         │   │                         │
│ [리포트 보기]             │   │ [멘토링 시작]            │   │                         │
└─────────────────────────┘   └─────────────────────────┘   └─────────────────────────┘
```

상태 계산은 `getCheckpointStatus()`에서 완료/진행중/첫CP/이전CP완료 순으로 판정.

## 8. MentoringChat — 채팅 UI

- **헤더**: CP명 + 진행률 (질문 N/M)
- **메시지 목록**: user/assistant 구분 렌더, SSE chunk 실시간 append
- **입력창**: 2000자 제한, Enter 전송
- **액션**: [세션 포기] / [리포트 보기]
- **훅**: `useMentoringSSE` (스트림 수신), `useMentoringSession` (React Query)

자유 채팅은 동일 컴포넌트를 `checkpointType='free_chat'`으로 재사용.

## 9. 리포트 & 피드백

**MentoringReport** (세션 완료 시 자동 생성된 `mentoring_reports` 렌더)
- 헤더: CP명 · 점수(0-100) · 생성일
- 본문: 구조화 JSON → 섹션별 렌더
- 추천 사항: `recommendations` 배열

**MentoringFeedback** (세션당 1회)
- ★ 1~5점 + 선택 코멘트
- 유니크 제약: `(session_id, user_id)`

War Room에 리포트 자동 공유 → 팀 전원 열람.

---

## 10. 관리자 제어 — 해커톤·팀 단위 2층 스위치

**해커톤 단위** — `hackathons.mentoring_mode` / `mentoring_config`
- 모드: `disabled` / `optional` / `required`
- 활성 CP 목록, 자유 채팅 on/off, LLM 모델, 일일 토큰 한도

**팀 단위** — `mentoring_enabled` 플래그 (2026-04-15 추가)
- 개별 팀별로 멘토링 기능 on/off 가능
- 해커톤 모드가 `enabled`여도 팀 플래그가 `false`면 진입 차단

**Office Hours** — `mentoring-office-hours.ts`
- 시간대 제한 운영 (예: 10:00~22:00만 AI 응답)

## 11. 데이터 모델 요약

| 테이블 | 핵심 필드 | 제약 |
|--------|-----------|------|
| `mentoring_sessions` | checkpointType, status, currentQuestionIndex, tokenUsage | 팀당 동일 CP `in_progress` 1개 |
| `mentoring_messages` | role, content, modelUsed, questionIndex, tokenCount | session FK cascade |
| `mentoring_reports` | reportType, contentJson, summaryText, score, recommendations | session FK cascade |
| `mentoring_feedback` | rating(1~5), comment | `(sessionId, userId)` unique |

`hackathons` 테이블에 `mentoringMode`, `mentoringConfig`, `mentoringEnabled` 컬럼 추가.

## 12. 비용·안전 가드레일

- **메시지 입력**: 2,000자 제한
- **팀당 일일 토큰**: 50,000 (관리자 조절 가능)
- **Rate Limit**: 메시지 10 req/min/user · 세션 5 req/min/team
- **타임아웃**: AI API 30초 → 자동 재시도 1회 → 사용자 안내
- **연속 실패**: 3회 이상 → 관리자 알림 + 멘토링 일시 중지
- **데이터 보존**: 대화 기록 90일 후 익명화 / 리포트 1년 후 통계만 보존
