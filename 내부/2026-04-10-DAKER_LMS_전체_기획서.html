    개요

    1.1 비전

    DAKER LMS는 AI 기반 맞춤형 학습 관리 시스템으로, 기업·대학·공공기관의 AI/데이터 교육을 위한 올인원 플랫폼이다. 단순 콘텐츠 전달을 넘어 AI 교재 자동 생성, 실시간 학습 분석, 게이미피케이션, 팀 학습 연계를 통합하여 학습 효과를 극대화한다.

    2026년 4월, Hybrid Plugin LMS 4-Layer 아키텍처를 도입하여 스테이지 타입 확장성, 이벤트 기반 분석, 통합 성적부 등 차세대 LMS 기능을 구현하고 있으며, 학습 그룹 관리, 일괄 등록 시스템, 경량 출결 시스템을 완성하여 대규모 조직 교육 운영 인프라를 갖추었다.

    Hybrid Plugin LMS 4-Layer 아키텍처

    2.1 왜 "Hybrid Plugin"인가

    해외 LMS 플랫폼(Moodle, Canvas, Coursera, Khan Academy 등)과 국내 플랫폼을 리서치한 결과, 두 가지 접근법이 존재했다:

    접근법장점단점대표
    모놀리식안정적, 일관된 UX타입 추가 시 대규모 수정 필요Coursera, Khan Academy
    플러그인 기반높은 확장성, 외부 개발자 참여복잡한 인터페이스, 버전 관리Moodle, Canvas

    DAKER는 단일팀 단일 코드베이스이므로 외부 플러그인 공개 필요 없이, 내부 개발 속도 향상을 위한 Hybrid 전략을 채택했다:

    • 플러그인 인터페이스는 내부용 (StagePlugin 계약)
    • 배포는 단일 번들 (기존 빌드/배포 파이프라인 유지)
    • 기존 동작을 한 줄도 깨지 않으면서 플러그인 구조를 덧씌움

    2.2 4-Layer 구조

    ┌─────────────────────────────────────────────────────────────┐
│ Layer 4 — 표준 호환 (SCORM/xAPI Export)                      │ ⏭️ 생략
├─────────────────────────────────────────────────────────────┤
│ Layer 3 — Assessment (Gradebook + 문제은행 + 과제)           │ 🔄 Phase 3-A 진행중
│   └─ Gradebook 서비스: plugin.score() → 통합 성적 집계       │
├─────────────────────────────────────────────────────────────┤
│ Layer 2 — 학습 이벤트 스트림 (xAPI 경량)                      │ ✅ Phase 1 완료
│   └─ learning_events 테이블 + 12 verbs + 7대 분석 지표       │
├─────────────────────────────────────────────────────────────┤
│ Layer 1 — Stage Plugin Core (스테이지 타입 확장 인프라)       │ ✅ Phase 2 완료
│   └─ 6종 플러그인 + 서버-클라이언트 대칭 레지스트리          │
└─────────────────────────────────────────────────────────────┘
                          ↑
                 기존 DAKER Learning Domain
              (교재/트랙/세션/스트릭/배지/AI생성)

    2.3 Layer 간 의존성 및 데이터 흐름

    사용자가 퀴즈 제출
  → LearningQuizViewer (Layer 1 quiz 플러그인)
  → plugin.toEvents() → eventStream.recordBatch() → learning_events (Layer 2)
  → plugin.score() → gradebookService 집계 (Layer 3)
  → 학습자 성적 뷰 / 관리자 Gradebook 대시보드
    Layer역할산출물다른 Layer와의 관계
    Layer 1스테이지 타입 플러그인StagePlugin 인터페이스, 레지스트리, 6종 플러그인Layer 2에 toEvents()로 이벤트 주입, Layer 3에 score() 제공
    Layer 2학습 이벤트 단일 소스learning_events 테이블, 분석 대시보드Layer 1의 toEvents() 결과를 수집, Layer 3의 원시 데이터
    Layer 3평가·성적 통합Gradebook 서비스, 문제 은행, 과제Layer 2 이벤트 집계 + Layer 1의 score() 호출
    Layer 4외부 표준 호환SCORM/xAPI export생략 (필요 시 Layer 2 변환으로 저비용 도입)

    2.4 Phase 순서 및 이유

    Phase 1 (Layer 2) ✅ 완료
         ↓
Phase 2 (Layer 1) ✅ 완료 ──→ toEvents() 계약으로 Phase 1 수집 훅 자동화
         ↓
Phase 3 (Layer 3) 🔄 진행중 ──→ 이벤트 + score() 조합으로 Gradebook 자연 구성

    Phase 1을 먼저 한 이유: Layer 2의 이벤트 스트림이 없으면 Layer 1의 toEvents()는 저장할 곳이 없고, Layer 3의 Gradebook은 집계할 단일 소스가 없다.

    콘텐츠 계층 구조

    DAKER LMS는 3-Tier 계층 구조로 학습 콘텐츠를 구성한다.

    3.1 계층도

    학습 트랙 (Track)
  └─ 코스 (Course) — 교재 모음
       └─ 교재 (Material) — 단일 학습 단위
            └─ 스테이지 (Stage) — 학습 활동 단위 (6종)

    3.2 각 계층 상세

    트랙 (Track)

    속성설명
    정의하나의 학습 목표를 달성하기 위한 코스·교재의 순서 있는 묶음
    유형industry (산업별), skill (기술별), custom (커스텀)
    진행 모드순차 강제, 순차 선택, 자유
    연계Raid/해커톤과 연계하여 참가자 사전 학습 경로로 활용
    AI 생성트랙 전체 커리큘럼 AI 자동 설계 + 일괄 교재 생성 지원

    코스 (Course)

    속성설명
    정의교재(Material)의 논리적 그룹. 하나의 주제 또는 모듈 단위
    진행 모드sequential_forced (순차 강제), sequential_optional (순차 선택), free (자유)
    상태draft, published, archived
    필수/선택트랙 내에서 필수 또는 선택 코스로 지정 가능

    교재 (Material)

    속성설명
    정의학습의 기본 단위. 여러 스테이지로 구성
    난이도beginner, intermediate, advanced
    상태draft → published → archived
    메타데이터제목, 설명, 썸네일, 카테고리, 태그, 예상 소요시간, 학습 목표
    AI 생성Gemini 기반 자동 생성 지원

    스테이지 (Stage)

    속성설명
    정의교재 내 개별 학습 활동. 화면 하나에 대응
    유형6종 (content, quiz, prompt, flashcard, pdf, branching-lesson)
    순서orderIndex로 정렬
    슬러그URL-friendly 식별자
    팁 콘텐츠학습 중 도움말로 표시되는 부가 설명

    항목설명
    유형quiz
    목적4지선다형 객관식 문제
    채점채점 가능 (80% 이상 통과)

    채점채점 가능 (유사도 점수 0~100)
    데이터 형식{ instruction, sampleAnswer, hasSampleAnswer }
    평가 항목의도(intent), 구조(structure), 명확성(clarity) 3축 평가
    시도 기록learningPromptAttempts 테이블에 사용자 프롬프트, AI 응답, 유사도, 피드백 저장
    뷰어LearningPromptViewer — 프롬프트 입력, AI 평가 결과 표시

    항목설명

    6.2 Gradebook 서비스 설계

    신규 DB 테이블 없이 기존 attempts 3종 + 진도 테이블을 Gradebook 서비스가 조합한다.

    interface GradebookEntry {
  stageId: string;
  stageTitle: string;
  stageType: string;          // 'quiz' | 'prompt' | 'flashcard' | 'content' | 'pdf'
  scorable: boolean;
  bestScore: number | null;    // 시도 중 최고 점수
  maxScore: number | null;     // 만점
  passed: boolean;             // 통과 여부 (plugin.score() 기준)
  attempts: number;            // 총 시도 횟수
  lastAttemptAt: Date | null;
  completed: boolean;          // 스테이지 완료 여부
}

interface MaterialGradebook {
  materialId: string;
  materialTitle: string;
  totalStages: number;
  completedStages: number;
  scorableStages: number;
  averageScore: number | null;  // scorable 스테이지 평균 점수율 (%)
  entries: GradebookEntry[];
}

    6.3 채점 로직 (plugin.score() 재활용)

    스테이지 유형점수 소스만점 기준통과 기준
    quizMAX(score) from learningQuizAttempts총 문제 수80% 이상
    promptMAX(similarityScore) from learningPromptAttempts100 (유사도 만점)plugin.score() 판정
    flashcardMAX(knownCount) from learningFlashcardAttempts총 카드 수passingThreshold 이상
    contentscorable: false—완료 여부만
    pdfscorable: false—완료 여부만

    각 서버 플러그인의 aiPromptFragment 속성에 해당 스테이지 유형의 데이터 형식이 한국어로 명시되어 있으며, AI 교재 파이프라인이 이를 자동으로 프롬프트에 포함한다.

    [학습 세션 로그 조회]

  durationSeconds < minDurationSeconds  → 'absent'
       │
       ▼

  detectionSource: 'auto'
